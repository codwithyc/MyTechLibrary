## 연산자

### 연산자란?
---
연산자는 값을 계산하거나 비교하거나, 조건을 조합할 때 사용하는 기호입니다.  
프로그래밍에서 값은 혼자 존재하는 경우보다 다른 값과 함께 계산되는 경우가 더 많습니다.

예를 들어 아래 코드를 보면 `+`, `>`, `&&`가 모두 연산자입니다.

```go
total := price + tax
isAdult := age >= 20
canEnter := isAdult && hasTicket
```

연산자를 이해할 때는 기호만 외우기보다 다음 세 가지를 함께 보는 것이 좋습니다.

- 어떤 타입의 값에 사용할 수 있는가?
- 결과로 어떤 타입의 값이 나오는가?
- 계산 순서는 어떻게 정해지는가?

Go는 타입을 엄격하게 구분하는 언어입니다.  
그래서 연산자도 "대충 비슷한 값이면 계산해준다"가 아니라, 타입이 맞아야 계산할 수 있습니다.

### 산술 연산자
---
산술 연산자는 숫자를 계산할 때 사용합니다.

| 연산자 | 의미 |
| --- | --- |
| `+` | 덧셈 |
| `-` | 뺄셈 |
| `*` | 곱셈 |
| `/` | 나눗셈 |
| `%` | 나머지 |

```go
package main

import "fmt"

func main() {
	a := 10
	b := 3

	fmt.Println(a + b)
	fmt.Println(a - b)
	fmt.Println(a * b)
	fmt.Println(a / b)
	fmt.Println(a % b)
}
```

정수끼리 나누면 결과도 정수입니다.

```go
fmt.Println(10 / 3) // 3
```

소수점까지 필요하다면 실수 타입을 사용해야 합니다.

```go
fmt.Println(10.0 / 3.0) // 3.3333333333333335
```

Go에서는 서로 다른 숫자 타입을 자동으로 섞어서 계산하지 않습니다.

```go
var a int = 10
var b float64 = 3.5

// fmt.Println(a + b) // 컴파일 에러
fmt.Println(float64(a) + b)
```

이런 엄격함은 처음에는 불편하지만, 값이 의도치 않게 바뀌는 실수를 줄여줍니다.

### 비교 연산자
---
비교 연산자는 두 값을 비교하고 결과로 `bool` 값을 만듭니다.  
즉 결과는 항상 `true` 또는 `false`입니다.

| 연산자 | 의미 |
| --- | --- |
| `==` | 같다 |
| `!=` | 다르다 |
| `<` | 작다 |
| `<=` | 작거나 같다 |
| `>` | 크다 |
| `>=` | 크거나 같다 |

```go
age := 20

fmt.Println(age == 20)
fmt.Println(age >= 19)
fmt.Println(age < 10)
```

비교 연산자는 조건문에서 특히 많이 사용됩니다.

```go
if age >= 20 {
	fmt.Println("성인입니다.")
}
```

문자열도 비교할 수 있습니다.

```go
fmt.Println("go" == "go")
fmt.Println("apple" < "banana")
```

문자열의 크기 비교는 사전식 순서에 가깝게 비교됩니다.  
다만 한글, 대소문자, 유니코드가 섞이면 사람이 기대하는 정렬과 다를 수 있으므로  
문자열 정렬은 나중에 별도 기능으로 다루는 것이 좋습니다.

### 실수 오차
---
실수는 겉보기에는 정확한 숫자처럼 보이지만, 컴퓨터 내부에서는 2진수로 근사해서 저장됩니다.  
그래서 일부 소수는 정확하게 표현되지 않습니다.

```go
package main

import "fmt"

func main() {
	a := 0.1
	b := 0.2

	fmt.Println(a + b)
	fmt.Println(a+b == 0.3)
}
```

`0.1 + 0.2`가 정확히 `0.3`이 되지 않을 수 있습니다.  
그래서 실수를 비교할 때는 `==`로 바로 비교하기보다 오차 범위를 두는 방식이 안전합니다.

```go
diff := (a + b) - 0.3
if diff < 0 {
	diff = -diff
}

fmt.Println(diff < 0.000001)
```

정리하면 다음과 같습니다.

- 정수 계산은 정확합니다.
- 실수 계산은 근사값이 섞일 수 있습니다.
- 돈, 포인트, 개수처럼 정확해야 하는 값은 가능하면 정수 단위로 표현하는 것이 좋습니다.

### 논리 연산자
---
논리 연산자는 `bool` 값을 조합할 때 사용합니다.

| 연산자 | 의미 |
| --- | --- |
| `&&` | 그리고 |
| `||` | 또는 |
| `!` | 부정 |

```go
age := 22
hasTicket := true

fmt.Println(age >= 20 && hasTicket)
fmt.Println(age < 20 || hasTicket)
fmt.Println(!hasTicket)
```

`&&`는 양쪽이 모두 `true`일 때만 `true`입니다.  
`||`는 둘 중 하나만 `true`여도 `true`입니다.  
`!`는 `true`를 `false`로, `false`를 `true`로 뒤집습니다.

Go의 논리 연산자는 필요한 만큼만 계산합니다.

```go
if user != nil && user.IsActive {
	fmt.Println("활성 사용자입니다.")
}
```

위 코드에서 `user != nil`이 `false`이면 뒤의 `user.IsActive`는 검사하지 않습니다.  
이런 방식을 단축 평가라고 합니다.

### 대입 연산자
---
대입 연산자는 변수에 값을 저장할 때 사용합니다.

```go
count := 10
count = 20
```

기존 값에 연산을 적용한 뒤 다시 저장하는 축약 표현도 있습니다.

```go
count += 5  // count = count + 5
count -= 2  // count = count - 2
count *= 3  // count = count * 3
count /= 2  // count = count / 2
count %= 4  // count = count % 4
```

1씩 증가하거나 감소할 때는 `++`, `--`를 사용할 수 있습니다.

```go
count++
count--
```

Go에서 `++`와 `--`는 표현식이 아니라 문장입니다.  
그래서 아래처럼 다른 계산 안에 끼워 넣을 수 없습니다.

```go
// fmt.Println(count++) // 컴파일 에러
count++
fmt.Println(count)
```

이 점은 C나 JavaScript와 다른 부분입니다.  
Go는 이런 문법을 단순하게 제한해서 읽기 어려운 코드를 줄입니다.

### 연산자 우선순위
---
연산자가 여러 개 섞이면 계산 순서가 중요합니다.

```go
result := 2 + 3 * 4
fmt.Println(result) // 14
```

곱셈이 덧셈보다 먼저 계산되므로 결과는 `20`이 아니라 `14`입니다.

우선순위를 모두 외우려고 하기보다, 헷갈릴 때는 괄호를 사용하는 것이 좋습니다.

```go
result := (2 + 3) * 4
fmt.Println(result) // 20
```

코드는 컴퓨터만 읽는 것이 아니라 사람이 계속 읽고 수정합니다.  
따라서 애매한 계산식은 괄호로 의도를 드러내는 편이 더 좋습니다.

### 핵심 요약
---
- 연산자는 값을 계산하거나 비교하거나 조건을 조합할 때 사용합니다.
- Go에서는 서로 다른 타입의 값을 마음대로 섞어 계산할 수 없습니다.
- 비교 연산자의 결과는 `bool`입니다.
- 실수는 근사값으로 저장될 수 있으므로 정확한 비교에 주의해야 합니다.
- 논리 연산자 `&&`, `||`는 단축 평가를 합니다.
- 복잡한 연산식은 괄호를 사용해 의도를 명확히 드러내는 것이 좋습니다.
