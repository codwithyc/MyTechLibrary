## 상수

### 상수란?
---
상수는 한 번 정하면 바뀌지 않는 값입니다.  
변수가 "변할 수 있는 값"이라면, 상수는 "변하지 않기로 약속한 값"입니다.

```go
const pi = 3.141592
```

상수는 프로그램 안에서 의미 있는 고정값을 표현할 때 사용합니다.

예를 들어 원의 넓이를 구할 때 `3.141592`라는 숫자를 코드 여러 곳에 직접 쓰면,  
나중에 이 값이 무엇을 의미하는지 알기 어렵습니다.

```go
area := radius * radius * 3.141592
```

상수에 이름을 붙이면 의미가 분명해집니다.

```go
const pi = 3.141592

area := radius * radius * pi
```

상수는 값을 바꾸지 못하게 막는 기능이기도 하지만,  
코드에 이름을 붙여 의미를 드러내는 기능이기도 합니다.

### 상수 선언
---
Go에서는 `const` 키워드로 상수를 선언합니다.

```go
const name = "Go"
const version = 1.22
const isOpen = true
```

타입을 명시할 수도 있습니다.

```go
const maxCount int = 100
const message string = "hello"
```

여러 상수를 묶어서 선언할 수도 있습니다.

```go
const (
	statusReady = 1
	statusRunning = 2
	statusDone = 3
)
```

상수는 선언과 동시에 값을 정해야 합니다.  
변수처럼 나중에 값을 넣을 수 없습니다.

```go
// const score int // 컴파일 에러
```

그리고 상수에는 `:=`를 사용할 수 없습니다.

```go
// pi := 3.14 // 변수 선언
const pi = 3.14 // 상수 선언
```

### 상수는 언제 사용하나?
---
상수는 다음 상황에서 유용합니다.

- 프로그램 전체에서 의미가 고정된 값
- 코드에 직접 적으면 의미가 불분명한 값
- 여러 곳에서 반복해서 사용하는 값
- 상태값이나 분류값처럼 정해진 목록

예를 들어 할인율을 상수로 둘 수 있습니다.

```go
const discountRate = 0.1

price := 10000
discount := float64(price) * discountRate
```

상태값도 상수로 표현할 수 있습니다.

```go
const (
	orderPending = 1
	orderPaid = 2
	orderCanceled = 3
)
```

숫자 `1`, `2`, `3`만 보면 의미를 알 수 없지만,  
`orderPaid`라는 이름을 보면 주문 결제 완료 상태라는 사실이 드러납니다.

### iota
---
Go에서는 연속된 상수값을 만들 때 `iota`를 사용할 수 있습니다.

```go
const (
	statusReady = iota
	statusRunning
	statusDone
)
```

이 코드는 다음과 같습니다.

```go
const (
	statusReady = 0
	statusRunning = 1
	statusDone = 2
)
```

`iota`는 상수 묶음 안에서 줄이 바뀔 때마다 0부터 1씩 증가합니다.

상태값을 직접 숫자로 관리하면 중간에 값을 추가하거나 순서를 바꿀 때 실수하기 쉽습니다.  
`iota`를 사용하면 연속된 값을 더 안전하게 만들 수 있습니다.

### 타입 없는 상수
---
Go의 상수는 타입이 있을 수도 있고 없을 수도 있습니다.  
이 부분은 Go 상수를 이해할 때 중요합니다.

```go
const number = 10
```

위 상수는 아직 구체적인 타입이 정해지지 않은 상수입니다.  
필요한 곳에서 문맥에 맞게 타입이 결정됩니다.

```go
var a int = number
var b int64 = number
var c float64 = number
```

변수라면 타입이 다른 곳에 그대로 넣을 수 없지만,  
타입 없는 상수는 값 자체가 표현 가능하면 여러 타입에 사용될 수 있습니다.

반면 타입을 명시한 상수는 해당 타입을 가집니다.

```go
const typedNumber int = 10

var a int = typedNumber
// var b int64 = typedNumber // 컴파일 에러
```

정리하면 다음과 같습니다.

- 타입 없는 상수는 문맥에 따라 타입이 정해집니다.
- 타입 있는 상수는 선언한 타입을 유지합니다.
- 상수는 컴파일 시점에 계산될 수 있는 값입니다.

### 상수와 리터럴
---
리터럴은 코드에 직접 적은 값입니다.

```go
10
"hello"
true
3.14
```

상수는 이런 리터럴에 이름을 붙인 것이라고 이해할 수 있습니다.

```go
const maxRetry = 3
const greeting = "hello"
```

하지만 모든 리터럴을 상수로 만들 필요는 없습니다.  
한 번만 쓰이고 의미가 분명한 값은 그대로 두어도 괜찮습니다.

상수를 사용하는 기준은 "이 값에 이름을 붙였을 때 코드가 더 잘 읽히는가?"입니다.

```go
if retryCount >= 3 {
	fmt.Println("재시도 횟수를 초과했습니다.")
}
```

위 코드보다 아래 코드가 의미를 더 분명하게 전달합니다.

```go
const maxRetry = 3

if retryCount >= maxRetry {
	fmt.Println("재시도 횟수를 초과했습니다.")
}
```

### 핵심 요약
---
- 상수는 변하지 않는 값을 이름으로 표현하는 방법입니다.
- `const` 키워드로 선언합니다.
- 상수는 선언과 동시에 값을 정해야 합니다.
- `iota`는 연속된 상수값을 만들 때 유용합니다.
- 타입 없는 상수는 사용되는 문맥에 따라 타입이 정해질 수 있습니다.
- 상수는 코드에서 값의 의미를 드러내는 데 도움이 됩니다.
