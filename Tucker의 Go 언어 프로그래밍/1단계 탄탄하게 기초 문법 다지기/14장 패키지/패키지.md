## 패키지

### 패키지란?
---
패키지는 Go 코드를 묶는 가장 기본적인 단위입니다.  
Go 파일은 반드시 어떤 패키지에 속해야 합니다.

1장에서 처음 작성한 코드도 패키지 선언으로 시작했습니다.

```go
package main
```

`package main`은 실행 가능한 프로그램을 만들 때 사용하는 특별한 패키지입니다.  
그리고 `fmt`처럼 다른 패키지를 가져와 사용할 수도 있습니다.

```go
import "fmt"
```

패키지를 사용하는 이유는 단순합니다.

- 관련 있는 코드를 묶을 수 있습니다.
- 코드를 여러 파일로 나눌 수 있습니다.
- 다른 코드에서 재사용할 수 있습니다.
- 프로그램 구조를 더 명확하게 만들 수 있습니다.

작은 예제는 파일 하나로 충분하지만, 프로그램이 커지면 패키지로 나누는 습관이 중요해집니다.

### 패키지 사용하기
---
Go에서 다른 패키지를 사용하려면 `import`를 사용합니다.

```go
import "fmt"
```

여러 패키지를 가져올 때는 괄호로 묶을 수 있습니다.

```go
import (
	"fmt"
	"math/rand"
	"time"
)
```

패키지 안의 이름을 사용할 때는 `패키지명.이름` 형태로 접근합니다.

```go
fmt.Println("Hello")
```

여기서 `fmt`는 패키지 이름이고, `Println`은 `fmt` 패키지에서 외부로 공개한 함수입니다.

Go는 사용하지 않는 패키지를 import하면 컴파일 에러를 냅니다.

```go
import "fmt"

func main() {
	// fmt를 사용하지 않으면 컴파일 에러
}
```

처음에는 엄격하게 느껴지지만, 사용하지 않는 의존성이 코드에 남지 않도록 도와주는 장점이 있습니다.

### Go 모듈
---
모듈은 Go 프로젝트를 관리하는 단위입니다.  
패키지가 코드를 묶는 단위라면, 모듈은 여러 패키지를 포함하는 프로젝트 단위라고 볼 수 있습니다.

모듈은 `go.mod` 파일로 관리됩니다.

```text
module example.com/myapp

go 1.22
```

새 프로젝트를 시작할 때는 보통 아래 명령어를 실행합니다.

```bash
go mod init example.com/myapp
```

그러면 현재 폴더에 `go.mod` 파일이 만들어집니다.

모듈이 필요한 이유는 다음과 같습니다.

- 프로젝트 이름을 정합니다.
- 외부 패키지 의존성을 관리합니다.
- 빌드와 실행 기준을 명확하게 합니다.
- 여러 패키지로 나뉜 프로젝트를 하나로 묶습니다.

2장에서 `go mod`를 따로 정리한 이유도 여기에 있습니다.  
처음에는 `go run main.go`로 충분하지만, 프로젝트가 커지면 모듈 기준으로 관리하는 방식이 자연스럽습니다.

### 패키지명과 패키지 외부 공개
---
Go에서는 이름의 첫 글자가 대문자인지 소문자인지에 따라 공개 여부가 결정됩니다.

```go
func PrintMessage() {
	fmt.Println("외부에서 사용할 수 있습니다.")
}

func printLog() {
	fmt.Println("같은 패키지 안에서만 사용합니다.")
}
```

- 대문자로 시작하는 이름
    다른 패키지에서 사용할 수 있습니다.
- 소문자로 시작하는 이름
    같은 패키지 안에서만 사용할 수 있습니다.

이 규칙은 함수뿐만 아니라 변수, 상수, 구조체, 구조체 필드에도 적용됩니다.

```go
type User struct {
	Name string
	age  int
}
```

`Name` 필드는 외부 패키지에서 접근할 수 있지만, `age` 필드는 같은 패키지 안에서만 접근할 수 있습니다.

Go에서는 `public`, `private` 같은 키워드가 따로 없습니다.  
이름의 첫 글자만으로 공개 범위를 표현합니다.  
단순하지만 Go다운 규칙입니다.

### 패키지 초기화
---
Go 패키지에는 `init()` 함수라는 특별한 함수가 있습니다.  
`init()` 함수는 패키지가 사용되기 전에 자동으로 실행됩니다.

```go
func init() {
	fmt.Println("패키지 초기화")
}
```

`init()` 함수는 직접 호출하지 않습니다.  
Go 런타임이 패키지 초기화 과정에서 자동으로 호출합니다.

실행 순서를 단순하게 보면 다음과 같습니다.

1. import한 패키지를 초기화합니다.
2. 패키지 수준 변수를 초기화합니다.
3. `init()` 함수를 실행합니다.
4. `main()` 함수를 실행합니다.

`init()`은 설정값 등록, 드라이버 등록처럼 패키지 초기화가 필요할 때 사용됩니다.  
하지만 너무 많은 로직을 `init()`에 넣으면 프로그램 흐름을 파악하기 어려워집니다.

처음에는 `init()`을 자주 쓰려고 하기보다,  
"main보다 먼저 실행되는 패키지 초기화 함수가 있다" 정도로 이해해두면 충분합니다.

### 패키지를 나눌 때 생각할 점
---
패키지는 폴더를 나누는 일이 아니라, 코드의 책임을 나누는 일입니다.  
단순히 파일이 길어졌다는 이유만으로 패키지를 나누면 오히려 복잡해질 수 있습니다.

패키지를 나눌 때는 다음 질문을 해볼 수 있습니다.

- 이 코드들은 같은 역할을 하는가?
- 다른 곳에서도 재사용할 가능성이 있는가?
- 패키지 이름만 보고 역할을 이해할 수 있는가?
- 외부로 공개해야 할 이름과 숨겨야 할 이름이 구분되는가?

Go에서는 패키지 이름을 짧고 명확하게 짓는 편을 선호합니다.  
예를 들어 `utils`처럼 의미가 넓은 이름보다는, 실제 역할을 드러내는 이름이 좋습니다.

### 실습 코드
---
이번 장의 실습 파일은 아래 위치에 작성했습니다.

```text
14장 패키지/실습 파일/packages.go
```

실행 방법은 다음과 같습니다.

```bash
go run packages.go
```

실제 패키지 분리는 여러 파일과 폴더를 함께 다뤄야 하므로,  
이번 실습에서는 표준 패키지 사용과 공개 이름 규칙을 중심으로 확인합니다.

### 핵심 요약
---
- Go 파일은 반드시 하나의 패키지에 속합니다.
- `package main`은 실행 가능한 프로그램을 만들 때 사용합니다.
- `import`로 다른 패키지를 가져올 수 있습니다.
- 모듈은 Go 프로젝트를 관리하는 단위이며 `go.mod` 파일로 표현됩니다.
- Go에서는 이름이 대문자로 시작하면 외부 패키지에 공개됩니다.
- `init()` 함수는 패키지 초기화 시 자동으로 실행됩니다.
- 패키지는 폴더 나누기가 아니라 책임 나누기입니다.
