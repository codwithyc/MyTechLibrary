## 변수

### 변수란?
---
프로그래밍에서 변수는 가장 자주 사용되는 기본 요소 중 하나입니다.  
값을 저장하고, 필요할 때 다시 꺼내 쓰고, 상황에 따라 다른 값으로 바꾸기 위해 사용합니다.

Go 역시 변수를 매우 중요하게 다루며, 타입을 명확하게 구분하는 언어이기 때문에  
변수를 이해하는 과정에서 **타입, 값의 범위, 타입 변환, 숫자 표현 방식**까지 함께 익히는 것이 중요합니다.

이번 장에서는 먼저 Go 코드를 실습하기 위한 `mod` 설정 방법을 간단히 살펴본 뒤,  
변수가 무엇인지 선언 방법, 변수의 범위, 숫자 표현까지 차례대로 정리해보겠습니다.


### mod 설정과 변수 실습 준비
---
1장의 내용인 "Hello Go World"에선 예제를 실행할 때 `go run` 명령어를 사용했습니다.  
`go run`은 작성한 Go 파일을 바로 실행해볼 수 있어서, Hello World 처럼 짧고 단순한 예제를 확인할 때 매우 편리합니다.

예를 들어 `hello.go` 파일이 있다면 아래처럼 바로 실행할 수 있습니다.

```
go run main.go
```

이 방식은 실행 자체에는 충분하지만, 프로젝트가 조금씩 커지기 시작하면 아쉬운 점이 생깁니다.  
파일이 늘어나고, 패키지를 나누고, 외부 라이브러리를 사용하게 되면 단순히 파일 하나를 실행하는 것  
만으로는 프로젝트를 체계적으로 관리하기 어려워지기 때문입니다.

그래서 Go에서는 보통 프로젝트를 시작할 때 먼저 모듈(Module)을 생성합니다.

모듈은 현재 프로젝트를 하나의 작업 단위로 관리하기 위한 기준이며,  
패키지 구조와 의존성, 그리고 프로젝트의 이름을 관리할 때 사용됩니다.

즉, 정리하자면 다음과 같습니다.

- `go run`
    작성한 Go 코드를 바로 실행할 때 사용합니다.
- `go mod init`
    현재 폴더를 하나의 Go 프로젝트로 관리하기 위한 시작 설정입니다.

다시 말해, `go run`은 **실행 명령어**이고,  
`go mod init`은 **프로젝트를 관리하기 위한 초기 설정 명령어**입니다.  
서로 역할이 다르기 때문에 둘 중 하나만 쓰는 개념이 아니라, 필요에 따라 함께 사용하게 됩니다.

터미널에서 프로젝트 폴더를 이동한 뒤 아래 명령어를 실행하면 됩니다.

```go
go mod init example/variables
```

명령어를 실행하면 go.mod 파일이 생성됩니다.

```go
module example/variables

go 1.24.0
```

이 파일은 프로젝트 이름과 Go 버전 같은 정보를 관리합니다.  
처음에는 단순히 실행만 하면 되는 것처럼 보일 수 있지만,  
프로젝트가 조금만 커져도 go.mod를 기준으로 관리하는 방식이 훨씬 자연스럽습니다.  

따라서 이번 장에서는 go run으로 실행만 해보는 수준을 넘어,  
go mod init으로 프로젝트를 먼저 준비한 뒤 그 안에서 변수를 학습하는 흐름으로 진행하겠습니다.  

### 변수 선언
---

변수를 사용하려면 먼저 선언을 해야 합니다.  
변수 선언은 "이 이름의 변수를 만들고, 어떤 타입의 값을 저장할지 정한다"는 의미입니다.

Go에서 가장 기본적인 선언 방식은 다음과 같습니다.

```
var 변수이름 타입
```

예를 들면 다음과 같습니다.

```go
var age int
var name string
var isStudent bool
```

값까지 함께 넣으면서 선언할 수도 있습니다.

```go
var age int = 20
var name string = "Go"
var isStudent bool = true
```

이처럼 Go에서는 변수의 이름과 타입이 함께 중요하게 다루어집니다.

> go에서 `var`란? 
> Go에서 `var`는 변수를 선언할 때 사용하는 키워드입니다.  
> 변수를 만들고, 이름과 타입을 정하며, 필요하면 초기값까지 함께 저장할 수 있습니다.
> 예를 들어 `var age int = 20`은 `age`라는 이름의 정수형 변수를 선언하고 값으로 20을 저장한 것입니다.
> var는 Go에서 변수를 만드는 가장 기본적인 방법이며, 값을 나중에 넣을 수도 있고 여러 변수를 한 번에 선언할 수도 있습니다.
> 다만 Go에서는 var 외에도 변수를 선언하는 다른 방법이 있으며, 이 부분은 뒤에서 자세히 다루겠습니다.

### 변수에 대해 더 알아보기
---

변수는 단순히 값을 저장하는 상자라고만 보면 부족합니다.  
변수를 더 정확히 이해하려면 이름, 값, 타입, 주소라는 관점으로 함께 보는 것이 좋습니다.

#### 변수의 4가지 속성
---

- 이름 : 변수를 구분하기 위한 식별자
- 값 : 안에 저장된 실제 데이터
- 타입 : 변수에 어떤 종류의 값이 들어가는지 나타내는 정보
- 주소 : 변수가 메모리 어디에 저장되어 있는지 나타내는 위치 정보

Go에서는 `&` 연산자를 사용해 변수의 주소를 확인할 수 있습니다.

또한 Go는 타입을 엄격하게 구분하는 강한 타입 언어입니다.  
즉, 정수형 변수에 문자열을 넣거나, 서로 다른 타입을 아무렇게나 섞어 사용할 수 없습니다.

숫자 타입 역시 여러 종류가 있습니다.
- 정수형 : `int`, `int8`, `int16`, `int32`, `int64`
- 부호 없는 정수형 : `uint`, `uint8`, `uint16`, `uin32`, `uint64`
- 실수형 : `float32`, `float64`
각 타입은 저장할 수 있는 값의 범위가 다르기 때문에 상황에 맞는 선택이 필요합니다.

### 변수 선언의 다른 형태
---

Go에서는 기본 선언 외에도 더 간단한 방식으로 변수를 선언할 수 있습니다.

값을 함께 넣으면 타입을 생략할 수 있습니다.
```go
var age = 20
var name = "Go"
var pi = 3.14
```

함수 안에서는 `:=`를 사용해 더 짧게 선언할 수도 있습니다.
```go
age := 20
name := "Go"
```

또한 여러 변수를 한 번에 선언하는 것도 가능합니다.
```go
var x, y int = 10, 20
name, age := "유찬", 20
```

### 타입 변환
---

Go는 서로 다른 타입으로 자동으로 바꿔주지 않는 경우가 많습니다.  
따라서 필요한 경우에는 직접 타입 변환을 해야 합니다.

```go
var num int = 10
var result float64 = float64(num)
```

타입 변환은 단순해 보이지만 주의할 점도 있습니다.
- 큰 범위의 값을 작은 범위의 타입으로 바꾸면 값이 손실될 수 있습니다.
- 실수를 정수로 바꾸면 소수점 아래 값이 사라집니다.
- 숫자와 문자열은 자동 변환되지 않습니다.
예를 들어 숫자 문자열로 바꾸고 싶을 때는 `strconv` 패키지를 사용합니다.

> `strconv`는 Go의 문자열 변환 패키지입니다.
> 이름도 string + convert에서 온 거라고 보시면 됩니다.
> 간단하게 말하자면 `strconv`는 **문자열과 다른 타입 사이를 변환할 때 사용하는 표준 패키지입니다.**

> 해당 내용은 추후 상세하게 따로 다루도록 하겠습니다.

### 변수의 범위
---

변수는 아무 곳에서나 사용할 수 있는 것이 아니라, 선언된 위치에 따라 사용할 수 있는 범위가 정해집니다.  
이것을 스코프(scope)라고 합니다.

- 함수 안에서 선언한 변수는 해당 함수 안에서만 사용할 수 있습니다.
- 함수 밖에서 선언한 변수는 여러 함수에서 사용할 수 있습니다.
- `if`, `for` 블록 안에서 선언한 변수는 그 블록 안에서만 사용할 수 있습니다.

또한 변수의 범위는 "어디서 사용할 수 있는가" 뿐만 아니라  
"타입이 얼마나 큰 값을 담을 수 있는가"라는 의미로도 자주 함계 생각하게 됩니다.

### 숫자 표현
---

Go에서 숫자를 여러 방식으로 표현할 수 있습니다.  
정수는 소수점이 없는 숫자이고, 실수는 소수점이 있는 숫자입니다.

```go
var a int = 10
var pi float64 = 3.14
```

또한 Go는 2진수, 8진수, 16진수 표현도 지원합니다.

```go
var binary int = 0b1010
var octal int = 0o12
var hex int = 0xA
```

실수는 겉보기에는 정확해 보여도 내부적으로는 근사값으로 저장되기 때문에  
계산 결과에 미세한 오차가 생길 수 있습니다.

따라서 실수를 사용할 때는 **정밀도 문제**를 항상 염두에 두는 것이 좋습니다.
