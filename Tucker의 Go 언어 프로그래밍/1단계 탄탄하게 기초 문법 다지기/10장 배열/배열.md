## 배열

### 배열이란?
---
배열은 같은 타입의 값을 여러 개 묶어서 저장하는 자료구조입니다.  
변수를 하나씩 따로 만들면 값이 많아질수록 관리하기 어려워집니다.

예를 들어 학생 점수 5개를 저장한다고 해보겠습니다.

```go
score1 := 90
score2 := 85
score3 := 72
score4 := 100
score5 := 64
```

이렇게 작성하면 점수가 많아질수록 변수 이름도 많아지고, 반복 처리도 어렵습니다.  
배열을 사용하면 하나의 이름 아래 여러 값을 묶을 수 있습니다.

```go
scores := [5]int{90, 85, 72, 100, 64}
```

배열을 이해할 때 중요한 점은 다음 두 가지입니다.

- 배열에는 같은 타입의 값만 들어갑니다.
- 배열의 길이는 배열 타입의 일부입니다.

즉 `[3]int`와 `[5]int`는 서로 다른 타입입니다.

### 배열 사용법
---
배열을 선언하는 기본 형태는 다음과 같습니다.

```go
var 배열이름 [길이]타입
```

예를 들어 정수 3개를 담는 배열은 다음처럼 선언합니다.

```go
var numbers [3]int
```

값을 지정하지 않으면 각 요소는 타입의 기본값으로 초기화됩니다.  
`int`의 기본값은 `0`입니다.

```go
fmt.Println(numbers) // [0 0 0]
```

배열의 각 값은 인덱스로 접근합니다.  
인덱스는 0부터 시작합니다.

```go
numbers[0] = 10
numbers[1] = 20
numbers[2] = 30
```

배열을 선언하면서 값을 바로 넣을 수도 있습니다.

```go
scores := [5]int{90, 85, 72, 100, 64}
```

길이를 직접 쓰지 않고 `...`로 컴파일러에게 맡길 수도 있습니다.

```go
scores := [...]int{90, 85, 72, 100, 64}
```

배열의 길이는 `len()` 함수로 확인합니다.

```go
fmt.Println(len(scores))
```

반복문과 함께 사용하면 배열 전체를 쉽게 처리할 수 있습니다.

```go
for i := 0; i < len(scores); i++ {
	fmt.Println(scores[i])
}
```

`range`를 사용하면 인덱스와 값을 함께 받을 수 있습니다.

```go
for index, score := range scores {
	fmt.Println(index, score)
}
```

값만 필요하고 인덱스가 필요 없다면 `_`를 사용합니다.

```go
for _, score := range scores {
	fmt.Println(score)
}
```

### 배열은 연속된 메모리
---
배열은 같은 타입의 값이 메모리에 연속으로 배치된 구조입니다.  
이 특징 때문에 인덱스를 사용해 빠르게 요소에 접근할 수 있습니다.

예를 들어 `[3]int` 배열이 있다면, 세 개의 `int` 값이 순서대로 나란히 저장됩니다.

```go
numbers := [3]int{10, 20, 30}
```

개념적으로는 다음과 같이 볼 수 있습니다.

```text
인덱스:  0   1   2
값:     10  20  30
```

배열의 인덱스 범위는 `0`부터 `len(배열)-1`까지입니다.  
범위를 벗어난 인덱스에 접근하면 오류가 발생합니다.

```go
// fmt.Println(numbers[3]) // 잘못된 접근
```

Go는 이런 실수를 엄격하게 막아줍니다.  
배열 범위를 벗어나는 접근은 프로그램의 안전성을 해칠 수 있기 때문입니다.

### 배열은 값 타입
---
Go에서 배열은 값 타입입니다.  
배열을 다른 변수에 대입하면 배열 전체가 복사됩니다.

```go
a := [3]int{1, 2, 3}
b := a

b[0] = 100

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [100 2 3]
```

`b`를 바꿔도 `a`는 바뀌지 않습니다.  
배열 전체가 복사되었기 때문입니다.

이 특징은 안전하지만, 배열이 매우 크면 복사 비용이 커질 수 있습니다.  
실제 Go 코드에서는 길이가 고정된 작은 데이터에는 배열을 사용하고,  
크기가 변하거나 자주 전달되는 데이터에는 다음 단계에서 배우는 슬라이스를 더 많이 사용합니다.

### 다중 배열
---
배열 안에 배열을 넣으면 다중 배열을 만들 수 있습니다.  
대표적으로 표나 좌표처럼 행과 열이 있는 데이터를 표현할 때 사용합니다.

```go
board := [2][3]int{
	{1, 2, 3},
	{4, 5, 6},
}
```

접근할 때는 인덱스를 두 번 사용합니다.

```go
fmt.Println(board[0][0]) // 1
fmt.Println(board[1][2]) // 6
```

반복문도 중첩해서 사용할 수 있습니다.

```go
for row := 0; row < len(board); row++ {
	for col := 0; col < len(board[row]); col++ {
		fmt.Print(board[row][col], " ")
	}
	fmt.Println()
}
```

다중 배열도 결국 배열입니다.  
길이가 타입에 포함되고, 값으로 복사된다는 특징은 그대로 유지됩니다.

### 핵심 요약
---
- 배열은 같은 타입의 값을 정해진 개수만큼 묶어 저장하는 자료구조입니다.
- 배열의 인덱스는 0부터 시작합니다.
- 배열의 길이는 배열 타입의 일부입니다.
- `[3]int`와 `[5]int`는 서로 다른 타입입니다.
- 배열은 메모리에 연속적으로 저장됩니다.
- Go에서 배열은 값 타입이므로 대입하거나 함수에 전달할 때 배열 전체가 복사됩니다.
- 실제 개발에서는 배열보다 슬라이스를 더 자주 사용하지만, 슬라이스를 이해하려면 배열의 개념이 먼저 필요합니다.
