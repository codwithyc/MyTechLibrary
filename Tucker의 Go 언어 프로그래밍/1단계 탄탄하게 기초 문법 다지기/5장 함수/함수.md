## 함수

### 함수란?
---
함수는 여러 줄의 코드를 하나의 이름으로 묶어두는 방법입니다.  
프로그램이 커질수록 모든 코드를 `main()` 함수 안에 한 줄씩 계속 작성하기는 어렵습니다.

함수를 사용하면 코드를 다음처럼 나눌 수 있습니다.

- 어떤 일을 하는 코드인지 이름으로 표현할 수 있습니다.
- 같은 코드를 여러 번 반복해서 쓰지 않아도 됩니다.
- 큰 문제를 작은 문제로 나누어 생각할 수 있습니다.
- 나중에 테스트하거나 수정하기 쉬워집니다.

Go 프로그램은 이미 함수에서 시작했습니다.

```go
func main() {
	fmt.Println("Hello Go World")
}
```

`main()` 역시 함수입니다.  
다만 `package main` 안의 `main()` 함수는 프로그램이 시작될 때 자동으로 호출되는 특별한 함수입니다.

### 함수 정의
---
Go에서 함수를 정의하는 기본 형태는 다음과 같습니다.

```go
func 함수이름(매개변수 목록) 반환타입 {
	실행할 코드
}
```

두 정수를 더하는 함수를 만들어보겠습니다.

```go
package main

import "fmt"

func add(a int, b int) int {
	return a + b
}

func main() {
	result := add(10, 20)
	fmt.Println(result)
}
```

`add()` 함수는 `int` 값 두 개를 받아서, `int` 값 하나를 반환합니다.

같은 타입의 매개변수가 이어질 때는 타입을 한 번만 적을 수도 있습니다.

```go
func add(a, b int) int {
	return a + b
}
```

Go에서는 반환값이 여러 개일 수도 있습니다.

```go
func divide(a, b int) (int, bool) {
	if b == 0 {
		return 0, false
	}

	return a / b, true
}
```

이 함수는 나눗셈 결과와 성공 여부를 함께 반환합니다.  
Go에서는 이런 방식이 자주 사용됩니다. 특히 나중에 에러를 다룰 때 많이 보게 됩니다.

### 함수를 호출하면 생기는 일
---
함수를 사용하는 것을 함수 호출이라고 합니다.

```go
result := add(10, 20)
```

함수를 호출하면 다음 일이 일어납니다.

1. 호출한 쪽에서 값을 전달합니다.
2. 함수의 매개변수에 전달값이 복사됩니다.
3. 함수 본문이 실행됩니다.
4. `return`을 만나면 결과를 돌려주고 함수가 끝납니다.
5. 호출한 쪽은 반환값을 받아 계속 실행합니다.

중요한 점은 기본적으로 값이 복사되어 전달된다는 것입니다.

```go
func change(n int) {
	n = 100
}

func main() {
	age := 20
	change(age)
	fmt.Println(age) // 20
}
```

`change()` 안에서 `n`을 바꿔도 `age`는 바뀌지 않습니다.  
`age`의 값이 `n`으로 복사되었을 뿐, 같은 변수를 공유한 것이 아니기 때문입니다.

나중에 포인터를 배우면 함수 안에서 원래 변수의 값을 바꾸는 방법도 알게 됩니다.

### 함수는 왜 쓰나?
---
함수를 쓰는 이유는 단순히 코드를 짧게 만들기 위해서만은 아닙니다.  
함수의 가장 큰 장점은 생각을 나눌 수 있다는 점입니다.

예를 들어 숫자가 짝수인지 확인하는 코드가 있다고 해보겠습니다.

```go
number := 10

if number%2 == 0 {
	fmt.Println("짝수입니다.")
}
```

이 코드를 함수로 만들면 의미가 더 분명해집니다.

```go
func isEven(number int) bool {
	return number%2 == 0
}
```

이제 코드는 다음처럼 읽힙니다.

```go
if isEven(number) {
	fmt.Println("짝수입니다.")
}
```

함수 이름이 코드의 설명 역할을 하게 됩니다.  
좋은 함수 이름은 주석보다 더 강력할 때가 많습니다.

함수를 잘 나누려면 다음 기준을 생각하면 좋습니다.

- 한 함수는 하나의 역할에 집중하는가?
- 함수 이름만 보고 무엇을 하는지 알 수 있는가?
- 너무 많은 매개변수를 받지는 않는가?
- 결과를 예측하기 쉬운가?

### 재귀 호출
---
재귀 호출은 함수가 자기 자신을 다시 호출하는 방식입니다.

가장 대표적인 예시는 팩토리얼입니다.

```text
5! = 5 * 4 * 3 * 2 * 1
```

Go 코드로 작성하면 다음과 같습니다.

```go
func factorial(n int) int {
	if n <= 1 {
		return 1
	}

	return n * factorial(n-1)
}
```

재귀 함수에는 반드시 종료 조건이 있어야 합니다.

```go
if n <= 1 {
	return 1
}
```

이 조건이 없다면 함수가 끝나지 않고 계속 자기 자신을 호출합니다.  
재귀는 문제를 작게 쪼개기 좋은 방식이지만, 반복문보다 이해하기 어려울 수 있으므로  
처음에는 "종료 조건"과 "문제가 작아지는 방향"을 꼭 확인해야 합니다.

### 핵심 요약
---
- 함수는 코드를 이름이 있는 작은 단위로 묶는 방법입니다.
- `func` 키워드로 함수를 정의합니다.
- 매개변수는 함수가 외부에서 받는 값이고, 반환값은 함수가 돌려주는 결과입니다.
- Go 함수는 여러 값을 반환할 수 있습니다.
- 기본적으로 함수에 전달되는 값은 복사됩니다.
- 재귀 함수는 자기 자신을 호출하는 함수이며 반드시 종료 조건이 필요합니다.
