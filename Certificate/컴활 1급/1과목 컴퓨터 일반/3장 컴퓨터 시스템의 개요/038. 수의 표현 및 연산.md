## 038. 수의 표현 및 연산

### 핵심 요약

- 컴퓨터는 내부적으로 2진법을 사용한다.
- 2진수 3자리는 8진수 1자리와 대응한다.
- 2진수 4자리는 16진수 1자리와 대응한다.
- 10진수 → 다른 진수: 나누기 후 나머지를 역순으로 읽는다.
- 다른 진수 → 10진수: 자리값을 곱해서 더한다.
- 1의 보수: 비트 반전
- 2의 보수: 1의 보수 + 1
- 왼쪽 Shift: 2배 효과
- 오른쪽 Shift: 1/2 효과
- AND, OR, NOT은 대표적인 논리 연산이다.

### 1. 진법

컴퓨터는 내부적으로 **2진법(Binary)** 을 사용하여 모든 연산을 수행합니다.  
또한 컴퓨터를 학습할 때는 2진수 외에도 **8진수(Octal)** 와 **16진수(Hexadecimal)** 를 자주 사용합니다.

| 진법 | 설명 |
| --- | --- |
| **2진법(Binary)** | 0과 1, 두 개의 숫자로 표현합니다. |
| **8진법(Octal)** | 0부터 7까지의 숫자로 표현하며, **2진수 3자리를 묶어서** 하나의 수로 표현할 수 있습니다. |
| **10진법(Decimal)** | 0부터 9까지의 숫자로 표현합니다. |
| **16진법(Hexadecimal)** | 0부터 9까지의 숫자와, 10부터 15를 의미하는 **A ~ F** 문자로 표현합니다. |

### 2. 진법 변환

컴퓨터는 내부적으로 **2진수(Binary)** 를 사용하여 데이터를 처리합니다.  
반면 사람은 보통 **10진수(Decimal)** 를 사용하므로, 컴퓨터를 공부할 때는 각 진법 사이의 변환 방법을 이해하는 것이 중요합니다.

진법 변환은 크게 다음과 같이 나눌 수 있습니다.

- **10진수를 다른 진수로 변환하는 방법**
- **2진수, 8진수, 16진수 사이를 서로 변환하는 방법**

#### 2-1. 10진수를 2진수, 8진수, 16진수로 변환

10진수를 다른 진수로 변환할 때는 **해당 진수로 계속 나누고, 나머지를 역순으로 읽습니다.**

- **10진수 → 2진수**: 2로 나눈 나머지를 아래에서 위로 읽음
- **10진수 → 8진수**: 8로 나눈 나머지를 아래에서 위로 읽음
- **10진수 → 16진수**: 16으로 나눈 나머지를 아래에서 위로 읽음  
  (단, 나머지가 10~15이면 A~F로 표현)

예를 들어, **10진수 13**을 2진수로 변환하면 다음과 같습니다.

- 13 ÷ 2 = 6 ... 1
- 6 ÷ 2 = 3 ... 0
- 3 ÷ 2 = 1 ... 1
- 1 ÷ 2 = 0 ... 1

따라서 **10진수 13 = 2진수 1101** 입니다.

#### 2-2. 2진수, 8진수, 16진수를 10진수로 변환

각 자리의 숫자에 **자리값(가중치)** 을 곱한 후 모두 더하면 됩니다.

- **2진수**는 각 자리에 \(2^0\), \(2^1\), \(2^2\), ... 의 값을 가집니다.
- **8진수**는 각 자리에 \(8^0\), \(8^1\), \(8^2\), ... 의 값을 가집니다.
- **16진수**는 각 자리에 \(16^0\), \(16^1\), \(16^2\), ... 의 값을 가집니다.

예를 들어, **2진수 1101**을 10진수로 변환하면 다음과 같습니다.

- \(1 \times 2^3 + 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0\)
- \(= 8 + 4 + 0 + 1 = 13\)

따라서 **2진수 1101 = 10진수 13** 입니다.

#### 2-3. 2진수와 8진수의 변환

2진수와 8진수는 쉽게 변환할 수 있습니다.  
그 이유는 **8 = 2³** 이므로, **2진수 3자리와 8진수 1자리**가 서로 대응하기 때문입니다.

- **2진수 → 8진수**: 오른쪽부터 3자리씩 묶음
- **8진수 → 2진수**: 각 자리를 3자리 2진수로 변환

예를 들어,

- **2진수 101110** → **101 / 110** → **5 / 6** → **8진수 56**

#### 2-4. 2진수와 16진수의 변환

2진수와 16진수도 쉽게 변환할 수 있습니다.  
그 이유는 **16 = 2⁴** 이므로, **2진수 4자리와 16진수 1자리**가 서로 대응하기 때문입니다.

- **2진수 → 16진수**: 오른쪽부터 4자리씩 묶음
- **16진수 → 2진수**: 각 자리를 4자리 2진수로 변환

예를 들어,

- **2진수 11010110** → **1101 / 0110** → **D / 6** → **16진수 D6**

#### 2-5. 소수점이 있는 진수 변환

정수 부분은 나누기 방법을 사용하고,  
소수 부분은 **해당 진수를 계속 곱하면서 정수 부분을 순서대로 읽는 방식**을 사용합니다.

예를 들어, 10진수 소수 **0.625**를 2진수로 변환하면 다음과 같습니다.

- 0.625 × 2 = 1.25 → 정수부 1
- 0.25 × 2 = 0.5 → 정수부 0
- 0.5 × 2 = 1.0 → 정수부 1

따라서 **0.625(10) = 0.101(2)** 입니다.

> **핵심 정리**
>
> - 10진수 → 다른 진수: 나누기 후 나머지를 역순으로 읽음
> - 다른 진수 → 10진수: 각 자리수에 자리값을 곱해서 더함
> - 2진수 3자리 = 8진수 1자리
> - 2진수 4자리 = 16진수 1자리

### 3. 보수

보수는 어떤 수를 쉽게 계산하거나, 특히 **뺄셈을 덧셈으로 바꾸어 처리하기 위해 사용하는 방법**입니다.  
컴퓨터에서는 뺄셈을 직접 수행하기보다 **보수를 이용하여 덧셈으로 처리하는 방식**을 많이 사용합니다.

보수에는 크게 **진의 보수**와 **감의 보수**가 있습니다.

- **10진수**
  - 10의 보수
  - 9의 보수
- **2진수**
  - 2의 보수
  - 1의 보수

#### 3-1. 10의 보수와 9의 보수

어떤 10진수에 대하여,

- **9의 보수**: 각 자리의 숫자를 9에서 뺀 값
- **10의 보수**: 9의 보수에 1을 더한 값

예를 들어, **325**의 보수는 다음과 같습니다.

- **9의 보수**: 674  
  (9-3=6, 9-2=7, 9-5=4)
- **10의 보수**: 675  
  (9의 보수 674 + 1)

즉,

- **10의 보수 = 10ⁿ - 원래 수**
- **9의 보수 = (10ⁿ - 1) - 원래 수**

여기서 \(n\)은 자리 수를 의미합니다.

#### 3-2. 2의 보수와 1의 보수

2진수에서는 다음과 같이 구합니다.

- **1의 보수**: 각 비트를 반전  
  (0 → 1, 1 → 0)
- **2의 보수**: 1의 보수에 1을 더한 값

예를 들어, **1010**의 보수는 다음과 같습니다.

- **1의 보수**: 0101
- **2의 보수**: 0110

컴퓨터에서는 음수를 표현하거나 뺄셈을 처리할 때 **2의 보수**를 많이 사용합니다.

#### 3-3. 보수를 사용하는 이유

컴퓨터는 기본적으로 **덧셈 회로**를 중심으로 연산을 처리합니다.  
따라서 뺄셈을 직접 처리하기보다, **보수를 이용하여 덧셈 형태로 바꾸면 회로 구성이 단순해지고 연산 효율이 높아집니다.**

예를 들어,

- \(A - B\)
- \(A + B의 보수\)

와 같이 바꾸어 계산할 수 있습니다.

#### 3-4. 2의 보수의 특징

2의 보수는 컴퓨터에서 매우 중요하게 사용됩니다.

- 양수와 음수를 하나의 덧셈 회로로 처리할 수 있습니다.
- 0이 하나만 존재합니다.
- 부호 비트를 포함하여 연산하기 편리합니다.

반면, 1의 보수 방식은 0이 **+0, -0** 두 개로 표현될 수 있어 비효율적입니다.  
그래서 실제 컴퓨터에서는 **2의 보수 방식**이 주로 사용됩니다.

#### 3-5. 보수의 예

예를 들어, 8비트에서 **5의 2의 보수**를 구하면 다음과 같습니다.

- 5 = `00000101`
- 1의 보수 = `11111010`
- 2의 보수 = `11111011`

즉, `11111011`은 8비트 기준에서 **-5**를 나타냅니다.

> **핵심 정리**
>
> - 9의 보수: 각 자리를 9에서 뺀 값
> - 10의 보수: 9의 보수 + 1
> - 1의 보수: 각 비트를 반전
> - 2의 보수: 1의 보수 + 1
> - 컴퓨터는 뺄셈을 보수를 이용해 덧셈으로 처리한다.
> - 실제 컴퓨터에서는 주로 **2의 보수**를 사용한다.

### 4. 연산

#### 4-1. 연산의 종류

| 구분 | 설명 |
| --- | --- |
| **단항(Unary) 연산** | 연산에 참여하는 데이터가 1개인 연산입니다. 예: Move, Shift, Rotate, Complement, NOT |
| **이항(Binary) 연산** | 연산에 참여하는 데이터가 2개인 연산입니다. 예: AND, OR, XOR, 사칙연산 |
| **수치적 연산** | 사칙연산, 산술 Shift 등 수를 대상으로 하는 연산입니다. |
| **비수치적 연산** | AND, OR, NOT, 논리 Shift, Rotate, Move 등 논리나 비트 처리를 위한 연산입니다. |

#### 4-2. 수치적 연산(산술 Shift)

##### 왼쪽 Shift

- 비트를 왼쪽으로 1칸 이동하면 **원래 값에 2를 곱한 것과 같은 효과**가 있습니다.
- 따라서 곱셈 연산에 활용됩니다.
- Shift 결과가 저장 공간의 크기를 초과하거나 부호가 바뀌면 **넘침(Overflow)** 이 발생할 수 있습니다.

##### 오른쪽 Shift

- 비트를 오른쪽으로 1칸 이동하면 **원래 값을 2로 나눈 것과 같은 효과**가 있습니다.
- 따라서 나눗셈 연산에 활용됩니다.
- 오른쪽으로 밀려나가는 값이 유효 비트이면 **잘림(Truncation)** 이 발생할 수 있습니다.

#### 4-3. 비수치적 연산

| 연산 | 설명 |
| --- | --- |
| **AND 연산** | 이항 연산으로, 데이터의 일부를 선택하거나 불필요한 비트를 제거할 때 사용됩니다. |
| **OR 연산** | 이항 연산으로, 데이터의 일부 비트를 추가하거나 설정할 때 사용됩니다. |
| **NOT 연산** | 단항 연산으로, 입력된 값의 보수를 취합니다. (0 → 1, 1 → 0) |

> **핵심 정리**
>
> - 단항 연산: 데이터 1개 사용
> - 이항 연산: 데이터 2개 사용
> - 왼쪽 Shift: 2를 곱하는 효과
> - 오른쪽 Shift: 2로 나누는 효과
> - AND, OR, NOT은 대표적인 비수치적 연산이다.
