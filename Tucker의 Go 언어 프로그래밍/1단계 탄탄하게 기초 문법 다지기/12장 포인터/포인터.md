## 포인터

### 포인터란?
---
포인터는 값이 저장된 메모리 주소를 담는 변수입니다.  
처음 들으면 어렵게 느껴지지만, 2장에서 변수를 이름, 값, 타입, 주소로 나누어 봤다면 조금 더 자연스럽게 이해할 수 있습니다.

변수는 메모리 어딘가에 값을 저장합니다.

```go
age := 20
```

이때 `age`는 값 `20`을 가지고 있고, 동시에 메모리 어딘가에 위치합니다.  
그 위치를 확인할 때 `&` 연산자를 사용합니다.

```go
fmt.Println(&age)
```

`&age`는 "age 변수의 주소"라는 뜻입니다.  
이 주소를 저장하는 변수가 포인터입니다.

```go
var p *int
p = &age
```

`*int`는 `int` 값을 가리키는 포인터 타입입니다.

### 포인터 기본 사용
---
포인터를 이해할 때는 두 연산자를 구분해야 합니다.

- `&`
    변수의 주소를 가져옵니다.
- `*`
    포인터가 가리키는 위치의 값을 가져오거나 바꿉니다.

```go
package main

import "fmt"

func main() {
	age := 20
	p := &age

	fmt.Println(age)
	fmt.Println(p)
	fmt.Println(*p)

	*p = 30
	fmt.Println(age)
}
```

`p`는 `age`의 주소를 가지고 있습니다.  
`*p`는 "p가 가리키는 곳에 있는 값"을 의미합니다.

따라서 `*p = 30`은 다음처럼 읽을 수 있습니다.

```text
p가 가리키는 변수의 값을 30으로 바꿔라.
```

결과적으로 `age`의 값이 바뀝니다.

### 포인터는 왜 쓰나?
---
Go에서 함수에 값을 전달하면 기본적으로 값이 복사됩니다.

```go
func change(n int) {
	n = 100
}

func main() {
	age := 20
	change(age)
	fmt.Println(age) // 20
}
```

`change()` 함수 안의 `n`은 `age`의 복사본입니다.  
그래서 `n`을 바꿔도 원래 변수인 `age`는 바뀌지 않습니다.

원래 변수의 값을 바꾸고 싶다면 주소를 전달해야 합니다.

```go
func change(n *int) {
	*n = 100
}

func main() {
	age := 20
	change(&age)
	fmt.Println(age) // 100
}
```

포인터를 사용하는 대표적인 이유는 다음과 같습니다.

- 함수 안에서 원래 변수의 값을 바꾸고 싶을 때
- 큰 구조체를 매번 복사하지 않고 전달하고 싶을 때
- 어떤 값이 없음을 `nil`로 표현하고 싶을 때
- 여러 코드가 같은 데이터를 공유해야 할 때

하지만 포인터를 무조건 많이 쓰는 것이 좋은 것은 아닙니다.  
값을 복사하는 편이 더 단순하고 안전한 경우도 많습니다.  
Go에서는 필요한 곳에만 포인터를 사용하고, 단순한 값은 값 그대로 다루는 스타일이 좋습니다.

### nil 포인터
---
포인터의 기본값은 `nil`입니다.  
`nil`은 "아무것도 가리키고 있지 않다"는 뜻입니다.

```go
var p *int
fmt.Println(p) // nil
```

`nil` 포인터가 가리키는 값을 읽으려고 하면 문제가 발생합니다.

```go
var p *int
// fmt.Println(*p) // 런타임 오류
```

포인터를 사용할 때는 `nil`인지 확인하는 습관이 중요합니다.

```go
if p != nil {
	fmt.Println(*p)
}
```

### 인스턴스
---
인스턴스는 어떤 타입을 바탕으로 실제로 만들어진 값입니다.  
예를 들어 `User`라는 구조체 타입이 있다면, 실제 사용자 값 하나하나가 인스턴스입니다.

```go
type User struct {
	Name string
	Age  int
}

user := User{Name: "gopher", Age: 10}
```

여기서 `User`는 타입이고, `user`는 그 타입으로 만들어진 인스턴스입니다.

구조체 인스턴스의 주소를 포인터로 다룰 수 있습니다.

```go
p := &user
fmt.Println(p.Name)
```

Go에서는 구조체 포인터로 필드에 접근할 때 `(*p).Name`처럼 쓰지 않고  
`p.Name`처럼 간단히 쓸 수 있습니다. 컴파일러가 알아서 처리해줍니다.

```go
p.Age = 20
fmt.Println(user.Age) // 20
```

구조체 포인터를 사용하면 함수 안에서 구조체 값을 직접 수정할 수 있습니다.

```go
func birthday(user *User) {
	user.Age++
}
```

### 스택 메모리와 힙 메모리
---
프로그램이 값을 저장할 때 대표적으로 스택과 힙이라는 메모리 영역을 사용합니다.

- 스택
    함수 호출과 함께 만들어지고, 함수가 끝나면 정리되는 메모리 영역입니다.
- 힙
    함수가 끝난 뒤에도 살아 있어야 하는 값을 저장하는 메모리 영역입니다.

Go에서는 개발자가 직접 스택과 힙을 선택하지 않습니다.  
컴파일러가 값의 사용 방식을 보고 적절한 위치를 결정합니다.

예를 들어 함수 안에서 만든 값의 주소를 밖으로 반환하면,  
그 값은 함수가 끝난 뒤에도 살아 있어야 합니다.

```go
func newUser(name string) *User {
	user := User{Name: name}
	return &user
}
```

이런 경우 Go 컴파일러는 `user`가 안전하게 살아 있도록 처리합니다.  
따라서 C 언어처럼 지역 변수 주소를 반환하면 무조건 위험하다고 외울 필요는 없습니다.

중요한 것은 다음입니다.

- Go에서는 포인터를 사용할 수 있지만 포인터 연산은 허용하지 않습니다.
- 메모리 해제는 가비지 컬렉터가 관리합니다.
- 포인터는 값을 공유하거나 수정할 때 사용하는 도구입니다.

### 핵심 요약
---
- 포인터는 값이 저장된 메모리 주소를 담는 변수입니다.
- `&변수`는 변수의 주소를 가져옵니다.
- `*포인터`는 포인터가 가리키는 값을 의미합니다.
- 함수에 포인터를 전달하면 함수 안에서 원래 값을 수정할 수 있습니다.
- 포인터의 기본값은 `nil`입니다.
- Go는 포인터를 제공하지만 포인터 연산은 허용하지 않아 비교적 안전하게 사용할 수 있습니다.
