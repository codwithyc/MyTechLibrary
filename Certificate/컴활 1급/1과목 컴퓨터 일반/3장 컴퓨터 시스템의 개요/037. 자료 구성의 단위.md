## 037. 자료 구성의 단위

자료 구성의 단위는 작은 것에서 큰 순서로 다음과 같습니다.

**비트 → 니블 → 바이트 → 워드 → 필드 → 레코드 → 블록 → 파일 → 데이터베이스**

| 단위 | 핵심 내용 |
| --- | --- |
| **비트(Bit)** | 정보 표현의 최소 단위, 0과 1로 표현 |
| **니블(Nibble)** | 4비트, 16진수 1자리 표현 |
| **바이트(Byte)** | 8비트, 문자 표현의 최소 단위, 1Byte = 256(2^8)가지 정보 표현 |
| **워드(Word)** | CPU가 한 번에 처리할 수 있는 단위 |
| **필드(Field)** | 의미 있는 정보의 최소 단위, 파일 구성의 최소 단위 |
| **레코드(Record)** | 관련된 필드의 집합 |
| **블록(Block)** | 하나 이상의 레코드가 모인 입출력 단위 |
| **파일(File)** | 여러 레코드의 집합 |
| **데이터베이스(Database)** | 여러 관련 파일의 집합 |
