## Go 문서 읽기와 함수 시그니처 추가 설명

함수를 배우기 시작하면 이제 직접 함수를 만들기도 하지만, 표준 라이브러리의 함수를 읽는 일도 많아집니다.  
Go를 제대로 공부하려면 함수 문법만큼이나 **문서에서 함수 시그니처를 읽는 능력**이 중요합니다.

이번 추가 설명은 함수를 더 많이 만드는 방법이 아니라, Go 문서를 읽고 함수 사용법을 파악하는 방법을 정리합니다.

### 함수 시그니처는 사용 설명서다
---

함수 시그니처는 함수 이름, 매개변수, 반환값을 보여줍니다.

```go
func Atoi(s string) (int, error)
```

이 시그니처만 보고도 다음을 알 수 있습니다.

- 함수 이름은 `Atoi`입니다.
- 문자열 하나를 입력으로 받습니다.
- 정수와 에러를 함께 반환합니다.

Go 문서를 볼 때는 본문 설명을 읽기 전에 시그니처를 먼저 보는 습관이 좋습니다.  
시그니처만 정확히 읽어도 함수 사용법의 절반은 이해한 것입니다.

### 반환값이 여러 개면 실패 가능성을 먼저 본다
---

Go 표준 라이브러리에는 `(값, error)` 형태의 함수가 많습니다.

```go
value, err := strconv.Atoi("123")
```

이런 함수는 실패할 수 있다는 뜻입니다.  
문서를 읽을 때 반환값에 `error`가 보이면 반드시 에러 처리 흐름까지 같이 생각해야 합니다.

```go
value, err := strconv.Atoi("abc")
if err != nil {
    return err
}

fmt.Println(value)
```

즉, Go에서 함수 사용법을 읽는다는 것은 "성공하면 무엇을 주는가"뿐 아니라  
"실패하면 어떻게 알려주는가"까지 읽는 것입니다.

### pkg.go.dev는 표준 라이브러리 사전이다
---

Go 공식 패키지 문서는 보통 `pkg.go.dev`에서 확인할 수 있습니다.

예를 들어 `fmt.Println`이 궁금하다면 `fmt` 패키지 문서를 보고,  
문자열 변환 함수가 궁금하다면 `strconv` 패키지 문서를 확인합니다.

문서를 볼 때는 다음 순서로 읽으면 좋습니다.

1. 패키지 설명을 읽습니다.
2. 자주 쓰는 함수 목록을 봅니다.
3. 함수 시그니처를 확인합니다.
4. 예제 코드가 있으면 함께 실행해봅니다.

Go 표준 라이브러리는 문서와 예제가 비교적 잘 정리되어 있습니다.  
처음부터 외부 블로그만 찾기보다 공식 문서를 읽는 습관을 들이면 좋습니다.

### go doc 명령으로 터미널에서 문서를 볼 수 있다
---

인터넷 브라우저를 열지 않고도 터미널에서 문서를 확인할 수 있습니다.

```bash
go doc fmt.Println
```

패키지 전체 문서를 보고 싶다면 다음처럼 실행할 수 있습니다.

```bash
go doc fmt
```

함수를 공부할 때 `go doc`을 함께 사용하면, 예제 코드를 작성하다가 바로 함수 설명을 확인할 수 있습니다.

### 공개 함수와 비공개 함수도 문서에서 구분된다
---

Go에서는 이름이 대문자로 시작하면 패키지 밖에서 사용할 수 있습니다.

```go
func Add(a int, b int) int
```

소문자로 시작하는 함수는 같은 패키지 내부에서만 사용할 수 있습니다.

```go
func add(a int, b int) int
```

이 규칙은 문서와도 연결됩니다.  
패키지 사용자에게 보여줄 함수라면 이름, 주석, 시그니처가 모두 사용법이 됩니다.

### 정리
---

함수 장을 공부할 때는 함수를 직접 만드는 것만큼, 이미 만들어진 함수를 읽는 능력도 중요합니다.

정리하면 다음과 같습니다.

- 함수 시그니처는 함수의 사용 설명서입니다.
- 반환값에 `error`가 있으면 실패 가능성을 함께 읽어야 합니다.
- `pkg.go.dev`는 Go 패키지 문서를 확인하는 기본 장소입니다.
- `go doc`으로 터미널에서 문서를 볼 수 있습니다.
- 대문자로 시작하는 함수는 패키지 밖에 공개되는 API입니다.

Go를 공부하면서 모르는 함수가 나오면 먼저 시그니처를 읽어보는 습관을 들이면 좋습니다.
