## gofmt와 코드 스타일 추가 설명

상수 장에서는 `const` 블록처럼 여러 값을 한 번에 선언하는 코드를 자주 보게 됩니다.  
이때 Go 코드가 일정한 모양으로 정리되는 이유는 대부분 `gofmt` 덕분입니다.

이번 추가 설명은 상수 문법 자체가 아니라, Go를 공부하면서 반드시 익숙해져야 하는 코드 정리 도구와 스타일을 다룹니다.

### Go에는 사실상 하나의 기본 포맷이 있다
---

Go는 코드 스타일 논쟁을 줄이기 위해 `gofmt`라는 공식 포맷터를 제공합니다.

```bash
gofmt -w main.go
```

`-w` 옵션은 포맷 결과를 파일에 바로 반영합니다.

Go 프로젝트에서는 개인 취향에 따라 들여쓰기나 정렬 방식을 크게 바꾸는 문화가 강하지 않습니다.  
대부분 `gofmt`가 정리한 모양을 그대로 따릅니다.

### const 블록도 gofmt가 보기 좋게 정리한다
---

상수를 여러 개 선언할 때는 보통 블록으로 묶습니다.

```go
const (
    StatusOK      = 200
    StatusCreated = 201
    StatusBadRequest = 400
)
```

`gofmt`를 실행하면 탭과 간격이 정리됩니다.

```go
const (
    StatusOK         = 200
    StatusCreated    = 201
    StatusBadRequest = 400
)
```

직접 공백을 맞추려 애쓰기보다, `gofmt`에게 맡기는 것이 Go다운 방식입니다.

### 저장할 때 자동 포맷을 켜두면 좋다
---

Go를 공부할 때는 에디터에서 저장 시 자동 `gofmt`가 실행되도록 설정해두는 것이 좋습니다.

이렇게 하면 다음 장점이 있습니다.

- 코드 모양 때문에 고민할 시간이 줄어듭니다.
- 예제 코드가 항상 일관된 형태로 유지됩니다.
- Git diff에서 의미 없는 공백 변경이 줄어듭니다.
- 다른 Go 코드와 스타일이 자연스럽게 맞습니다.

Go 학습 초반에는 문법도 익숙하지 않은데 포맷까지 직접 맞추려 하면 피로도가 커집니다.  
자동 포맷은 그 부담을 줄여줍니다.

### gofmt는 코드를 고쳐주지만 설계를 고쳐주지는 않는다
---

`gofmt`는 들여쓰기, 정렬, 줄바꿈 같은 형식을 정리합니다.  
하지만 변수 이름이 모호하거나, 함수가 너무 길거나, 구조가 어색한 문제까지 해결해주지는 않습니다.

```go
func f(a int, b int) int {
    return a + b
}
```

이 코드는 `gofmt`로 정리해도 함수 이름 `f`가 무엇을 의미하는지는 알기 어렵습니다.

즉, `gofmt`는 코드의 모양을 정리하고, 좋은 이름과 좋은 구조는 개발자가 고민해야 합니다.

### gofmt와 gofmt -w의 차이
---

`gofmt`만 실행하면 포맷 결과를 화면에 출력합니다.

```bash
gofmt main.go
```

파일을 실제로 수정하려면 `-w` 옵션을 사용합니다.

```bash
gofmt -w main.go
```

여러 파일을 한 번에 정리하고 싶다면 패키지 단위나 폴더 단위로 실행할 수도 있습니다.

```bash
gofmt -w .
```

### 정리
---

Go를 공부할 때 `gofmt`는 선택 도구가 아니라 기본 습관에 가깝습니다.

정리하면 다음과 같습니다.

- Go 코드는 보통 `gofmt`가 정리한 스타일을 따릅니다.
- `const` 블록처럼 여러 선언이 있는 코드도 자동으로 정렬됩니다.
- 에디터에서 저장 시 자동 포맷을 켜두면 학습이 편해집니다.
- `gofmt`는 형식을 정리하지만 이름과 구조까지 좋게 만들지는 않습니다.
- 파일에 바로 반영하려면 `gofmt -w`를 사용합니다.

Go를 익힐 때는 "내 스타일로 맞추기"보다 "Go가 정한 기본 스타일에 익숙해지기"가 더 중요합니다.
