## 표준 스트림과 터미널 실행 흐름 추가 설명

3장에서 `fmt` 패키지를 배우면 자연스럽게 "출력한다", "입력받는다"는 표현을 사용하게 됩니다.  
그런데 Go 프로그램이 실제로 어디에 출력하고, 어디에서 입력을 받는지 이해하면 `fmt`를 훨씬 정확하게 볼 수 있습니다.

이번 추가 설명은 `fmt` 문법 자체보다, Go 프로그램이 터미널과 연결되는 기본 흐름을 정리합니다.

### 프로그램은 기본 입출력 통로를 가진다
---

터미널에서 실행되는 프로그램은 보통 세 가지 기본 통로를 가집니다.

- 표준 입력, stdin  
  프로그램이 값을 읽어오는 기본 통로입니다.

- 표준 출력, stdout  
  프로그램이 정상적인 결과를 내보내는 기본 통로입니다.

- 표준 에러, stderr  
  오류나 진단 메시지를 내보내는 기본 통로입니다.

`fmt.Println()`은 기본적으로 표준 출력으로 값을 보냅니다.  
`fmt.Scan()`은 기본적으로 표준 입력에서 값을 읽습니다.

즉, `fmt`는 단순히 화면과 직접 연결된 도구라기보다, 프로그램의 기본 입출력 통로를 사용하게 해주는 패키지라고 볼 수 있습니다.

### 화면 출력과 stdout은 같은 말이 아니다
---

보통은 stdout이 터미널 화면에 연결되어 있기 때문에 `fmt.Println()` 결과가 화면에 보입니다.

```bash
go run main.go
```

하지만 stdout은 파일로 바꿔 연결할 수도 있습니다.

```bash
go run main.go > result.txt
```

이렇게 실행하면 프로그램의 출력이 화면이 아니라 `result.txt` 파일에 저장됩니다.  
Go 코드가 바뀐 것이 아니라, 실행할 때 출력 통로를 다르게 연결한 것입니다.

이 개념을 알면 CLI 프로그램을 만들 때 출력과 로그를 구분해야 하는 이유도 이해하기 쉬워집니다.

### 입력도 키보드만 의미하지 않는다
---

`fmt.Scan()`은 사용자가 키보드로 입력한 값을 읽는 것처럼 보입니다.

하지만 stdin 역시 다른 입력으로 바꿔 연결할 수 있습니다.

```bash
go run main.go < input.txt
```

이 경우 프로그램은 키보드가 아니라 `input.txt`에서 값을 읽습니다.

그래서 Go 프로그램 입장에서 중요한 것은 "키보드인가 파일인가"가 아니라,  
표준 입력 통로로 어떤 데이터가 들어오는가입니다.

### 파이프는 프로그램끼리 연결한다
---

터미널에서는 한 프로그램의 출력을 다른 프로그램의 입력으로 연결할 수 있습니다.

```bash
cat input.txt | go run main.go
```

이때 `cat input.txt`의 stdout이 Go 프로그램의 stdin으로 연결됩니다.

이런 흐름은 Go로 CLI 도구를 만들 때 매우 중요합니다.  
프로그램이 stdin에서 읽고 stdout으로 쓰도록 만들면, 다른 명령어와 조합하기 쉬운 도구가 됩니다.

### stderr는 오류 메시지를 분리할 때 사용한다
---

정상 결과는 stdout으로, 오류 메시지는 stderr로 보내는 것이 일반적입니다.

```go
fmt.Fprintln(os.Stderr, "입력값이 올바르지 않습니다")
```

이렇게 하면 정상 출력과 오류 출력을 구분할 수 있습니다.

```bash
go run main.go > result.txt 2> error.log
```

위 명령은 정상 결과는 `result.txt`에, 에러 메시지는 `error.log`에 저장합니다.  
작은 예제에서는 잘 느껴지지 않지만, 실제 도구를 만들 때는 중요한 차이입니다.

### 정리
---

`fmt` 패키지를 배울 때는 함수 이름만 외우기보다 프로그램의 입출력 통로를 함께 이해하는 것이 좋습니다.

정리하면 다음과 같습니다.

- `fmt.Println()`은 보통 stdout으로 출력합니다.
- `fmt.Scan()`은 보통 stdin에서 입력을 읽습니다.
- stdout은 화면이 아니라 다른 파일이나 프로그램으로 연결될 수 있습니다.
- stdin도 키보드가 아니라 파일이나 다른 프로그램 출력이 될 수 있습니다.
- 오류 메시지는 stderr로 분리하면 CLI 도구답게 동작합니다.

이 관점을 가지면 `fmt`는 단순 출력 함수가 아니라, Go 프로그램이 외부 세계와 대화하는 가장 기본적인 입출력 도구로 보입니다.
