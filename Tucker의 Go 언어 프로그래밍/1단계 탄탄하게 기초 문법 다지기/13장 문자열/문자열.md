## 문자열

### 문자열이란?
---
문자열은 여러 문자를 이어 붙인 값입니다.  
Go에서 문자열 타입은 `string`입니다.

```go
message := "Hello Go"
```

문자열은 화면에 출력할 문장, 사용자 이름, 파일 경로, 로그 메시지처럼  
프로그램에서 매우 자주 사용됩니다.

Go의 문자열을 이해할 때는 다음 두 가지를 구분해야 합니다.

- 바이트
    실제 메모리에 저장되는 단위입니다.
- 문자
    사람이 읽는 글자 단위입니다.

영어 알파벳은 보통 한 글자가 1바이트로 표현되지만,  
한글 같은 문자는 UTF-8에서 여러 바이트로 표현됩니다.

```go
fmt.Println(len("Go"))
fmt.Println(len("고"))
```

`len()`은 문자열의 글자 수가 아니라 바이트 수를 반환합니다.  
이 점이 Go 문자열에서 매우 중요합니다.

### 문자열 선언
---
문자열은 큰따옴표로 선언합니다.

```go
name := "gopher"
```

줄바꿈이나 탭 같은 특수 문자를 포함할 수 있습니다.

```go
message := "Hello\nGo"
```

백틱을 사용하면 여러 줄 문자열을 그대로 표현할 수 있습니다.

```go
text := `Hello
Go
Language`
```

큰따옴표 문자열은 `\n`, `\t` 같은 이스케이프 문자를 해석합니다.  
백틱 문자열은 대부분의 문자를 있는 그대로 보존합니다.

### 문자열 순회
---
문자열을 반복문으로 순회할 수 있습니다.

먼저 인덱스로 접근하면 바이트 단위로 접근합니다.

```go
word := "Go"

for i := 0; i < len(word); i++ {
	fmt.Println(word[i])
}
```

`word[i]`의 타입은 `byte`입니다.  
영어 문자열에서는 큰 문제가 없어 보이지만, 한글 문자열에서는 주의가 필요합니다.

```go
word := "고"

for i := 0; i < len(word); i++ {
	fmt.Println(word[i])
}
```

한글 한 글자는 UTF-8에서 여러 바이트로 저장되기 때문에,  
인덱스로 순회하면 글자가 아니라 바이트 조각을 보게 됩니다.

문자 단위로 순회하려면 `range`를 사용합니다.

```go
word := "고 Go"

for index, r := range word {
	fmt.Printf("index: %d, rune: %c\n", index, r)
}
```

`range`로 문자열을 순회하면 유니코드 문자 단위인 `rune`을 얻을 수 있습니다.  
여기서 `index`는 글자 번호가 아니라 해당 문자가 시작하는 바이트 위치입니다.

### 문자열 합치기
---
문자열은 `+` 연산자로 합칠 수 있습니다.

```go
first := "Hello"
second := "Go"

message := first + " " + second
fmt.Println(message)
```

간단한 문자열 연결에는 `+`를 사용해도 좋습니다.  
하지만 반복문 안에서 문자열을 계속 더하면 매번 새 문자열이 만들어져 비효율적일 수 있습니다.

```go
result := ""

for i := 0; i < 1000; i++ {
	result += "a"
}
```

이런 경우에는 `strings.Builder`를 사용하는 것이 좋습니다.

```go
var builder strings.Builder

for i := 0; i < 1000; i++ {
	builder.WriteString("a")
}

result := builder.String()
```

처음에는 `+`로 문자열을 합치는 방법을 익히고,  
반복적으로 많은 문자열을 만들 때 `strings.Builder`를 떠올리면 됩니다.

### 문자열 구조
---
Go의 문자열은 읽기 전용 바이트 배열에 가까운 구조로 이해할 수 있습니다.  
문자열 값은 실제 데이터와 길이 정보를 가지고 있습니다.

```go
text := "hello"
```

이 문자열은 내부적으로 다음 정보를 가진다고 볼 수 있습니다.

- 문자열 데이터가 있는 위치
- 문자열의 바이트 길이

그래서 `len(text)`는 빠르게 계산할 수 있습니다.  
다만 이 길이는 문자 수가 아니라 바이트 수입니다.

문자 수가 필요하다면 `utf8.RuneCountInString()`을 사용할 수 있습니다.

```go
count := utf8.RuneCountInString("고 Go")
fmt.Println(count)
```

문자열을 다룰 때는 "내가 지금 바이트를 다루는지, 문자를 다루는지"를 구분해야 합니다.

### 문자열은 불변이다
---
Go의 문자열은 불변입니다.  
즉 한 번 만들어진 문자열의 특정 위치를 직접 바꿀 수 없습니다.

```go
text := "hello"
// text[0] = 'H' // 컴파일 에러
```

문자열을 바꾸는 것처럼 보이는 코드는 실제로는 새 문자열을 만드는 것입니다.

```go
text := "hello"
text = "H" + text[1:]
```

문자열이 불변이면 다음 장점이 있습니다.

- 같은 문자열 데이터를 안전하게 공유할 수 있습니다.
- 함수에 전달해도 원본이 바뀔 걱정이 없습니다.
- 예측하기 쉬운 코드를 만들 수 있습니다.

문자열을 수정해야 한다면 `[]rune` 또는 `[]byte`로 변환한 뒤 처리하는 방법을 사용합니다.

```go
runes := []rune("hello")
runes[0] = 'H'
text := string(runes)

fmt.Println(text)
```

한글처럼 여러 바이트 문자가 섞일 수 있다면 `[]rune`을 사용하는 편이 안전합니다.

### 핵심 요약
---
- Go의 문자열 타입은 `string`입니다.
- `len()`은 문자열의 글자 수가 아니라 바이트 수를 반환합니다.
- 문자열을 인덱스로 접근하면 바이트 단위로 접근합니다.
- `range`를 사용하면 문자열을 `rune` 단위로 순회할 수 있습니다.
- 문자열은 불변이므로 특정 위치의 문자를 직접 바꿀 수 없습니다.
- 반복적으로 문자열을 합칠 때는 `strings.Builder`를 사용하는 것이 좋습니다.
