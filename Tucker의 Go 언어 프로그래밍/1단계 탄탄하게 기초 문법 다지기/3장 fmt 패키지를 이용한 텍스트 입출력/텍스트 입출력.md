## fmt 패키지를 이용한 텍스트 입출력

### fmt 패키지란?
---
프로그램은 혼자서만 동작하지 않습니다.  
사용자에게 결과를 보여주기도 하고, 사용자에게 값을 입력받기도 합니다.

Go에서 가장 기본적인 텍스트 입출력은 `fmt` 패키지를 통해 시작합니다.  
1장에서 사용했던 `fmt.Println()` 역시 이 패키지 안에 들어 있는 함수입니다.

`fmt`는 format의 줄임말로 볼 수 있습니다.  
단순히 글자를 출력하는 것뿐만 아니라, 값을 원하는 형식으로 바꿔 출력하거나  
키보드에서 입력받은 값을 변수에 저장하는 기능도 제공합니다.

이번 장에서는 다음 내용을 중심으로 정리합니다.

- 화면에 값을 출력하는 방법
- 출력 형식을 지정하는 방법
- 키보드 입력을 변수에 저장하는 방법
- `Scan()` 함수가 왜 변수의 주소를 필요로 하는지

### 표준 입출력
---
표준 입출력은 프로그램과 바깥세상이 약속해둔 기본 통로입니다.

- 표준 출력
    프로그램이 결과를 화면으로 내보내는 통로입니다.
- 표준 입력
    사용자가 키보드로 값을 입력하는 통로입니다.
- 표준 에러
    오류 메시지를 따로 내보내는 통로입니다.

처음 Go를 배울 때는 표준 출력과 표준 입력부터 익히면 충분합니다.  
터미널에서 프로그램을 실행하면 `fmt.Println()`으로 출력한 내용이 화면에 보이고,  
`fmt.Scan()`은 사용자가 터미널에 입력한 값을 읽어옵니다.

### 출력 함수
---
`fmt` 패키지에서 가장 자주 사용하는 출력 함수는 다음 세 가지입니다.

- `Print()`
    값을 그대로 출력합니다.
- `Println()`
    값을 출력한 뒤 줄바꿈을 합니다.
- `Printf()`
    형식 지정자를 사용해 원하는 형태로 출력합니다.

```go
package main

import "fmt"

func main() {
	name := "Go"
	version := 1.22

	fmt.Print("Hello ")
	fmt.Println(name)
	fmt.Printf("%s version %.2f\n", name, version)
}
```

실행 결과는 다음과 비슷합니다.

```text
Hello Go
Go version 1.22
```

`Println()`은 값을 여러 개 전달하면 값 사이에 공백을 넣고 마지막에 줄바꿈을 합니다.  
반면 `Printf()`는 문자열 안에 형식 지정자를 넣고, 뒤에 전달한 값을 그 자리에 끼워 넣습니다.

### 자주 사용하는 형식 지정자
---
`Printf()`에서 `%`로 시작하는 문자를 형식 지정자라고 합니다.  
형식 지정자는 "이 값을 어떤 모습으로 출력할지"를 알려주는 표시입니다.

| 형식 지정자 | 의미 | 예시 |
| --- | --- | --- |
| `%v` | 값의 기본 형태 | `fmt.Printf("%v", age)` |
| `%T` | 값의 타입 | `fmt.Printf("%T", age)` |
| `%d` | 10진수 정수 | `fmt.Printf("%d", age)` |
| `%f` | 실수 | `fmt.Printf("%f", height)` |
| `%.2f` | 소수점 둘째 자리까지 실수 | `fmt.Printf("%.2f", height)` |
| `%s` | 문자열 | `fmt.Printf("%s", name)` |
| `%t` | bool 값 | `fmt.Printf("%t", ok)` |
| `%p` | 주소값 | `fmt.Printf("%p", &age)` |

처음에는 전부 외울 필요는 없습니다.  
가장 먼저 `Println()`으로 값을 확인하고, 출력 모양을 맞춰야 할 때 `Printf()`를 사용하면 됩니다.

### 표준 입력
---
출력이 프로그램에서 사용자 방향으로 나가는 흐름이라면,  
입력은 사용자에게서 프로그램 안으로 들어오는 흐름입니다.

Go에서는 `fmt.Scan()`을 사용해 키보드 입력을 변수에 저장할 수 있습니다.

```go
package main

import "fmt"

func main() {
	var name string
	var age int

	fmt.Print("이름과 나이를 입력하세요: ")
	fmt.Scan(&name, &age)

	fmt.Println("이름:", name)
	fmt.Println("나이:", age)
}
```

터미널에서 아래처럼 입력할 수 있습니다.

```text
gopher 10
```

그러면 `name`에는 `"gopher"`가 들어가고, `age`에는 `10`이 들어갑니다.

### 왜 &를 붙여야 할까?
---
`fmt.Scan(&name, &age)`에서 가장 낯선 부분은 `&`입니다.  
`&`는 변수의 주소를 가져오는 연산자입니다.

입력 함수는 사용자가 입력한 값을 변수 안에 직접 넣어야 합니다.  
그런데 함수에 변수의 값만 전달하면, 함수는 원래 변수가 어디 있는지 알 수 없습니다.

그래서 `Scan()`에는 값 자체가 아니라 "이 변수에 저장해 주세요"라는 뜻으로  
변수의 주소를 넘겨야 합니다.

```go
var age int

fmt.Scan(&age)
```

이 코드는 다음처럼 읽을 수 있습니다.

```text
사용자가 입력한 값을 age 변수가 있는 위치에 저장해 주세요.
```

2장에서 변수를 이름, 값, 타입, 주소 관점으로 보자고 했던 이유가 여기서 바로 드러납니다.  
입력은 변수의 값을 바꾸는 일이기 때문에, 변수의 주소가 필요합니다.

### Scan, Scanf, Scanln
---
입력 함수도 여러 종류가 있습니다.

- `Scan()`
    공백이나 줄바꿈을 기준으로 값을 읽습니다.
- `Scanf()`
    형식 지정자에 맞춰 값을 읽습니다.
- `Scanln()`
    한 줄 안에서 값을 읽고, 줄 끝을 만나면 입력을 끝냅니다.

처음에는 `Scan()`만 정확히 이해해도 충분합니다.  
중요한 것은 입력받을 변수의 타입과 실제 입력값의 형태가 맞아야 한다는 점입니다.

```go
var age int

fmt.Scan(&age)
```

이 상태에서 숫자가 아니라 `hello` 같은 문자열을 입력하면 정수로 바꿀 수 없기 때문에  
정상적으로 값이 저장되지 않습니다.

실제로는 `Scan()`이 읽은 개수와 에러를 반환하므로, 아래처럼 확인하는 습관이 좋습니다.

```go
count, err := fmt.Scan(&age)
if err != nil {
	fmt.Println("입력 오류:", err)
	return
}

fmt.Println("읽은 값 개수:", count)
```

### 키보드 입력과 Scan 함수의 동작 원리
---
키보드로 값을 입력하면 값이 바로 변수에 들어가는 것처럼 보이지만,  
실제로는 다음 순서로 동작한다고 이해하면 좋습니다.

1. 프로그램이 입력을 기다립니다.
2. 사용자가 키보드로 값을 입력합니다.
3. 사용자가 Enter를 누르면 입력 내용이 프로그램으로 전달됩니다.
4. `Scan()`이 공백 또는 줄바꿈을 기준으로 값을 나눕니다.
5. 각 값을 전달받은 변수의 주소에 저장합니다.

즉, `Scan()`은 "키보드에서 들어온 텍스트를 읽어서, 지정한 변수에 맞는 타입으로 해석한 뒤 저장하는 함수"입니다.

이때 입력값과 변수 타입이 맞지 않으면 문제가 생깁니다.  
예를 들어 `int` 변수에는 정수 형태의 입력이 필요하고, `float64` 변수에는 실수 형태의 입력이 필요합니다.

### 핵심 요약
---
- `fmt` 패키지는 텍스트 출력과 입력을 다룰 때 사용하는 Go의 기본 패키지입니다.
- `Println()`은 값을 출력하고 줄바꿈을 합니다.
- `Printf()`는 형식 지정자를 사용해 출력 모양을 조절합니다.
- `Scan()`은 키보드 입력을 읽어 변수에 저장합니다.
- `Scan()`에 변수를 넘길 때는 값을 바꿀 수 있도록 변수의 주소인 `&변수명`을 전달합니다.
- 입력값의 형태와 변수 타입이 맞지 않으면 입력 오류가 발생할 수 있습니다.

### 마무리
---
이번 장에서는 프로그램이 사용자와 대화하는 가장 기본적인 방법을 배웠습니다.  
출력은 프로그램의 결과를 보여주는 일이고, 입력은 사용자에게서 값을 받아오는 일입니다.

앞으로 조건문과 반복문을 배우면, 입력받은 값에 따라 다른 결과를 만들 수 있습니다.  
그때부터 프로그램은 단순히 정해진 문장을 출력하는 수준을 넘어,  
사용자의 행동에 반응하는 형태로 발전하게 됩니다.
