## 041. 주기억장치

### 1. 주기억장치의 개요

주기억장치는 CPU가 직접 접근하여 데이터를 처리할 수 있는 기억장치(Memory)로,  
현재 수행 중인 프로그램과 데이터를 저장하고 있습니다.

- 종류에는 롬(ROM)과 램(RAM)이 있습니다.

#### 왜 주기억장치가 필요할까?

CPU는 매우 빠르게 동작하지만, 처리할 프로그램과 데이터가 전혀 없다면 아무 일도 할 수 없습니다.  
그래서 현재 실행에 필요한 명령어와 데이터를 가까운 곳에 올려 두는 공간이 필요한데, 그 역할을 하는 것이 주기억장치입니다.

즉, 주기억장치는  
**CPU가 바로 꺼내서 처리할 수 있도록 프로그램과 데이터를 임시로 올려 두는 공간**입니다.

#### 왜 보조기억장치가 아니라 주기억장치가 필요한가?

하드디스크나 SSD 같은 보조기억장치는 많은 데이터를 저장할 수 있지만, CPU가 직접 즉시 처리하기에는 상대적으로 느립니다.  
반면 주기억장치는 CPU가 직접 접근할 수 있어 실행 중인 작업을 빠르게 처리할 수 있습니다.

즉,

- **보조기억장치** : 오래 저장하는 공간
- **주기억장치** : 지금 실행할 내용을 올려 두는 공간

이라고 구분할 수 있습니다.

---

### 2. ROM(롬)

ROM(Read Only Memory)는 기억된 내용을 읽을 수만 있는 기억장치로서, 일반적으로는 쓰기가 불가능합니다.

- 전원이 꺼져도 기억된 내용이 지워지지 않는 **비휘발성 메모리**입니다.
- 주로 **펌웨어(Firmware)** 를 저장합니다.
- ROM에는 주로 **기본 입출력 시스템(BIOS)**, 글자 폰트, 자가진단 프로그램(POST : Power On Self Test) 등이 저장되어 있습니다.

#### 왜 ROM은 비휘발성 메모리일까?

컴퓨터가 꺼졌다가 다시 켜져도 반드시 남아 있어야 하는 정보들이 있습니다.  
예를 들어 컴퓨터가 처음 켜질 때 필요한 기본 동작 정보나 하드웨어를 점검하는 프로그램은 전원이 꺼졌다고 사라지면 안 됩니다.

그래서 이런 정보는 전원이 꺼져도 유지되는 **비휘발성 메모리**인 ROM에 저장합니다.

#### 왜 BIOS나 펌웨어를 ROM에 저장할까?

컴퓨터는 전원을 켜는 순간부터 바로 동작할 수 있어야 합니다.  
그런데 운영체제는 아직 불러오지 않은 상태이므로, 가장 먼저 실행될 기본 프로그램이 필요합니다.

이처럼 하드웨어를 초기화하고, 부팅을 준비하고, 기본 입출력을 담당하는 프로그램이 **펌웨어**이고, 이런 내용은 지워지면 안 되므로 ROM에 저장됩니다.

즉, ROM은  
**컴퓨터가 처음 시작할 때 반드시 필요한 기본 정보를 안정적으로 보관하는 공간**입니다.

#### ROM의 종류와 특징

ROM은 기억된 내용의 수정 가능 여부와 데이터 기록 방법에 따라 다음과 같이 분류됩니다.

| 종류 | 특징 |
| --- | --- |
| Mask ROM | 제조 과정에서 미리 내용을 기록한 ROM으로, 사용자가 임의로 수정할 수 없습니다. |
| PROM (Programmable ROM) | 특수 장비를 이용하여 한 번만 기록할 수 있으며, 이후에는 읽기만 가능합니다. |
| EPROM (Erasable PROM) | 자외선을 이용하여 기록된 내용을 여러 번 지우고 다시 기록할 수 있습니다. |
| EEPROM (Electrically Erasable PROM) | 전기적인 방법을 이용하여 기록된 내용을 여러 번 지우고 다시 기록할 수 있습니다. |

#### 왜 ROM도 여러 종류로 나뉠까?

모든 ROM이 완전히 같은 용도로 쓰이지는 않습니다.  
어떤 경우에는 제조할 때 내용을 고정해 두는 편이 좋고, 어떤 경우에는 나중에 내용을 수정할 수 있어야 합니다.

그래서

- 아예 수정이 불가능한 ROM도 있고
- 한 번만 기록 가능한 ROM도 있고
- 여러 번 지우고 다시 기록할 수 있는 ROM도 있는 것입니다.

즉, ROM은 단순히 “읽기 전용”이라는 공통점은 있지만,  
실제 사용 목적에 따라 수정 가능성이 다르게 설계됩니다.

### 3. RAM(램)

RAM(Random Access Memory)은 자유롭게 읽고 쓸 수 있는 기억장치입니다.

- RAM에는 현재 사용 중인 프로그램이나 데이터가 저장되어 있습니다.
- 전원이 꺼지면 기억된 내용이 모두 사라지는 **휘발성 메모리**입니다.
- 일반적으로 주기억장치라고 하면 램(RAM)을 의미합니다.
- 정보가 저장된 위치는 주소로 구분합니다.

#### 왜 RAM은 휘발성 메모리일까?

RAM은 영구 보관용이 아니라,  
현재 실행 중인 프로그램과 데이터를 잠시 올려 두고 빠르게 사용하기 위한 공간입니다.

즉, RAM의 핵심은 **빠른 접근과 빠른 처리**이지,  
오래 보존하는 것이 아닙니다.

그래서 전원이 꺼지면 내용이 사라지는 휘발성이라는 특징을 가집니다.

#### 왜 실행 중인 프로그램은 RAM에 올라와야 할까?

프로그램은 보통 SSD나 하드디스크에 저장되어 있습니다.  
하지만 CPU는 보조기억장치에 직접 접근해 고속으로 처리하기 어렵기 때문에,  
실행할 프로그램과 데이터를 먼저 RAM에 올려 놓고 처리합니다.

즉,

- 저장은 보조기억장치
- 실행은 RAM

이라는 흐름으로 이해하면 쉽습니다.

#### 왜 RAM은 주소로 구분할까?

RAM 안에는 매우 많은 데이터가 저장되므로, CPU가 필요한 데이터를 정확히 찾아가려면 각각의 저장 위치를 구분할 기준이 있어야 합니다.  
그 기준이 바로 **주소(Address)** 입니다.

즉, RAM의 각 저장 공간은 주소를 통해 식별되고, CPU는 주소를 이용해 원하는 위치에 접근합니다.

---

#### SRAM / DRAM의 특징

램(RAM)은 재충전 여부에 따라 DRAM(Dynamic RAM)과 SRAM(Static RAM)으로 분류됩니다.

| 구분 | 동적 램(DRAM) | 정적 램(SRAM) |
| --- | --- | --- |
| 구성 소자 | 콘덴서 | 플립플롭 |
| 특징 | 전원이 공급되어도 일정 시간이 지나면 전하가 방전되므로 주기적인 재충전이 필요함 | 전원이 공급되는 동안에는 기억 내용이 유지됨 |
| 전력 소모 | 적음 | 많음 |
| 접근 속도 | 느림 | 빠름 |
| 집적도(밀도) | 높음 | 낮음 |
| 가격 | 저가 | 고가 |
| 용도 | 일반적인 주기억장치 | 캐시 메모리 |

#### 왜 DRAM은 재충전이 필요할까?

DRAM은 데이터를 전하의 형태로 저장하는데, 시간이 지나면 전하가 자연스럽게 약해지거나 사라집니다.  
그래서 내용을 계속 유지하려면 일정 주기마다 다시 충전해 주어야 합니다.  
이 과정을 **재충전(Refresh)** 이라고 합니다.

즉, DRAM은 구조가 단순하고 집적도가 높지만,  
데이터 유지를 위해 주기적인 재충전이 필요합니다.

#### 왜 일반적인 주기억장치로는 DRAM을 사용할까?

DRAM은 SRAM보다 속도는 느리지만, 같은 면적에 더 많은 데이터를 저장할 수 있고 가격도 저렴합니다.  
그래서 큰 용량이 필요한 일반적인 주기억장치에는 DRAM이 더 적합합니다.

즉, 주기억장치는 보통 많은 용량이 필요하므로  
**가격과 집적도 측면에서 유리한 DRAM**을 주로 사용합니다.

#### 왜 SRAM은 캐시 메모리에 사용할까?

SRAM은 DRAM보다 비싸고 집적도는 낮지만 속도가 훨씬 빠릅니다.  
CPU는 아주 짧은 시간 안에 자주 사용하는 데이터를 빠르게 꺼내 써야 하므로, 속도가 가장 중요해집니다.

그래서 SRAM은 용량은 작더라도 빠른 접근이 필요한 **캐시 메모리**에 사용됩니다.

즉,

- **DRAM** : 크고 저렴해서 주기억장치에 적합
- **SRAM** : 빠르지만 비싸서 캐시에 적합

이라고 정리할 수 있습니다.

---

### 4. 기타 메모리

| 명칭 | 특징 |
| --- | --- |
| 캐시 메모리 (Cache Memory) | • 중앙처리장치(CPU)와 주기억장치 사이에 위치하여 컴퓨터의 처리 속도를 향상시키는 역할을 합니다. <br> • 캐시 메모리는 접근 속도가 빠른 정적 램(SRAM)을 사용하며, 용량은 주기억장치보다 작습니다. <br> • 캐시 적중률(Cache Hit Ratio)이 높을수록 시스템 전체 속도가 향상됩니다. <br> • L1 캐시(1차 캐시) : CPU 내부에 위치하며, 자주 사용하는 데이터를 저장하는 매우 빠른 기억공간입니다. <br> • L2 캐시(2차 캐시) : CPU가 필요로 할 가능성이 높은 데이터를 미리 저장해 두는 기억공간입니다. |
| 플래시 메모리 (Flash Memory) | • EEPROM의 일종인 비휘발성 메모리입니다. <br> • 스마트폰, USB 메모리, SSD, 디지털 카메라 등에 널리 사용됩니다. |
| 가상 메모리 (Virtual Memory) | • 보조기억장치의 일부를 주기억장치처럼 사용하는 메모리 관리 기법입니다. <br> • 주기억장치보다 큰 프로그램을 실행할 때 유용하게 사용됩니다. |
| 버퍼 메모리 (Buffer Memory) | • 두 장치가 데이터를 주고받을 때 속도 차이를 해결하기 위해 중간에 데이터를 임시 저장하는 공간입니다. <br> • 키보드 버퍼, 프린터 버퍼 등이 있으며, 캐시 메모리도 넓은 의미에서는 버퍼의 일종으로 볼 수 있습니다. |
| 연관 메모리 (Associative Memory) | • 기억장치에 저장된 정보에 접근할 때 주소 대신 기억된 내용의 일부를 이용하여 접근하는 장치입니다. <br> • 정보 검색이 빠르며, 캐시 메모리나 가상 메모리 매핑 테이블 등에 사용됩니다. |

#### 왜 캐시 메모리가 필요할까?

CPU는 매우 빠르지만, 주기억장치는 그보다 상대적으로 느립니다.  
그래서 CPU가 필요한 데이터를 매번 RAM에서 가져오면 속도 차이 때문에 대기 시간이 발생합니다.

이 문제를 줄이기 위해 CPU와 주기억장치 사이에 **더 빠른 작은 기억공간**을 두는데, 그것이 캐시 메모리입니다.

즉, 캐시 메모리는  
**자주 사용하는 데이터를 미리 가까운 곳에 저장해 두어 전체 처리 속도를 높이는 장치**입니다.

#### 왜 캐시 메모리는 SRAM을 사용할까?

캐시 메모리는 속도가 가장 중요합니다.  
주기억장치보다 훨씬 빠르게 동작해야 CPU가 기다리지 않기 때문입니다.

그래서 비싸고 용량은 작더라도 접근 속도가 빠른 **SRAM**을 사용합니다.

#### 왜 캐시 적중률이 중요할까?

CPU가 원하는 데이터가 캐시에 있으면 바로 가져올 수 있어 매우 빠릅니다.  
이것을 **캐시 적중(Cache Hit)** 이라고 합니다.

반대로 캐시에 없으면 RAM까지 가서 가져와야 하므로 느려집니다.  
이것을 **캐시 미스(Cache Miss)** 라고 합니다.

따라서 캐시 적중률이 높을수록 CPU가 빠르게 작업할 수 있어 전체 성능이 좋아집니다.

#### 왜 L1, L2처럼 캐시를 나눌까?

모든 캐시를 무조건 가장 빠르고 크게 만들면 좋겠지만, 실제로는 비용과 공간의 한계가 있습니다.  
그래서 속도와 용량의 균형을 위해 여러 단계로 나누어 구성합니다.

- **L1 캐시** : 가장 빠르지만 용량이 작음
- **L2 캐시** : L1보다 느리지만 더 큼

즉, 자주 쓰는 데이터일수록 더 가까운 캐시에 두고,  
덜 자주 쓰는 데이터는 그보다 조금 큰 캐시에 두는 방식입니다.

#### 왜 플래시 메모리가 많이 사용될까?

플래시 메모리는 전원이 꺼져도 데이터가 유지되는 비휘발성 메모리이면서도, 전기적으로 다시 쓰기가 가능합니다.  
그래서 저장장치로 활용하기에 매우 편리합니다.

즉, 플래시 메모리는  
**ROM처럼 데이터가 유지되면서도, RAM처럼 다시 기록이 가능하다는 장점**이 있어 다양한 기기에 널리 사용됩니다.

#### 왜 가상 메모리가 필요할까?

실행하려는 프로그램이 너무 크거나 동시에 실행되는 프로그램이 많으면, 실제 RAM만으로는 부족할 수 있습니다.  
이때 보조기억장치의 일부를 마치 RAM처럼 활용하여 부족한 주기억장치를 보완하는 방식이 가상 메모리입니다.

즉, 가상 메모리는  
**실제 주기억장치의 한계를 보완해 더 큰 프로그램을 실행할 수 있게 하는 기법**입니다.

다만 보조기억장치는 RAM보다 훨씬 느리므로, 가상 메모리를 많이 사용할수록 속도는 떨어질 수 있습니다.

#### 왜 버퍼 메모리가 필요할까?

두 장치가 데이터를 주고받을 때는 처리 속도가 서로 다를 수 있습니다.

예를 들어

- CPU는 매우 빠르고
- 프린터는 상대적으로 느리며
- 키보드는 사람이 입력하는 속도에 따라 느리게 들어옵니다.

이처럼 속도가 다른 장치들이 직접 데이터를 주고받으면 병목이 생기기 쉽습니다.  
그래서 중간에 데이터를 잠시 저장해 두는 공간이 필요한데, 그것이 버퍼 메모리입니다.

즉, 버퍼는  
**장치 간 속도 차이를 완화하기 위한 임시 저장 공간**입니다.

#### 왜 연관 메모리는 주소 대신 내용으로 찾을까?

일반적인 기억장치는 주소를 통해 접근하지만, 어떤 경우에는 “어디에 저장되었는지”보다 “무슨 내용인지”를 기준으로 빠르게 찾는 것이 더 중요합니다.

이럴 때 사용하는 것이 연관 메모리입니다.

즉, 연관 메모리는  
**주소 기반 접근이 아니라 내용 기반 탐색을 통해 검색 속도를 높이기 위한 방식**입니다.

---

### 한 줄 정리

주기억장치는 CPU가 직접 접근하여 현재 실행에 필요한 프로그램과 데이터를 저장하는 공간이며,  
ROM은 **지워지지 않아야 하는 기본 정보 저장용**, RAM은 **현재 실행을 위한 작업 공간**으로 사용됩니다.
