## 슬라이스

### 슬라이스
---
슬라이스는 배열의 일부를 가리키는 가변 길이 뷰입니다.  
배열은 길이가 타입에 포함되지만, 슬라이스는 길이를 유연하게 바꿔가며 사용할 수 있습니다.

```go
numbers := []int{10, 20, 30}
```

겉보기에는 배열과 비슷하지만 `[]int`는 `[3]int`와 전혀 다른 타입입니다.

- `[3]int`
    길이가 3으로 고정된 배열 타입입니다.
- `[]int`
    길이가 바뀔 수 있는 슬라이스 타입입니다.

실무에서 배열보다 슬라이스를 훨씬 자주 쓰는 이유는 다음과 같습니다.

- 요소 개수를 미리 고정하지 않아도 됩니다.
- 함수에 전달하기 편합니다.
- `append()`로 값을 자연스럽게 추가할 수 있습니다.
- 표준 라이브러리 대부분이 슬라이스를 기준으로 동작합니다.

슬라이스를 만드는 방법은 여러 가지가 있습니다.

```go
nums1 := []int{1, 2, 3}
nums2 := make([]int, 3)
nums3 := make([]int, 3, 5)
```

- `[]int{1, 2, 3}`
    값을 넣으면서 바로 생성합니다.
- `make([]int, 3)`
    길이 3, 용량 3인 슬라이스를 만듭니다.
- `make([]int, 3, 5)`
    길이 3, 용량 5인 슬라이스를 만듭니다.

여기서 길이와 용량을 구분하는 것이 중요합니다.

- `len()`
    현재 슬라이스에 포함된 요소 개수입니다.
- `cap()`
    현재 슬라이스가 같은 기반 배열에서 확장할 수 있는 최대 범위입니다.

```go
nums := make([]int, 3, 5)

fmt.Println(len(nums)) // 3
fmt.Println(cap(nums)) // 5
```

### 슬라이스 동작 원리
---
슬라이스는 데이터를 직접 모두 들고 있는 값이라기보다,  
"어느 배열의 어디부터 어디까지 볼 것인가"를 표현하는 작은 구조라고 이해하면 좋습니다.

개념적으로 슬라이스는 다음 정보를 가집니다.

- 시작 위치
- 길이
- 용량

예를 들어 아래 코드가 있다고 해보겠습니다.

```go
arr := [5]int{10, 20, 30, 40, 50}
slice := arr[1:4]
```

이 슬라이스는 `arr`의 일부를 바라봅니다.

```text
배열:    [10 20 30 40 50]
인덱스:    0  1  2  3  4
슬라이스:    [20 30 40]
```

중요한 점은 슬라이스가 배열을 복사해서 가지는 것이 아니라는 점입니다.  
그래서 슬라이스를 통해 값을 바꾸면 원본 배열도 함께 바뀝니다.

```go
arr := [5]int{10, 20, 30, 40, 50}
slice := arr[1:4]

slice[0] = 999

fmt.Println(arr)   // [10 999 30 40 50]
fmt.Println(slice) // [999 30 40]
```

이 특성 때문에 슬라이스는 효율적이지만,  
같은 기반 배열을 여러 슬라이스가 공유할 수 있다는 점도 항상 의식해야 합니다.

특히 `append()`는 상황에 따라 다르게 동작합니다.

- 남은 용량이 충분하면
    같은 기반 배열에 값을 이어 붙입니다.
- 남은 용량이 부족하면
    더 큰 새 배열을 만들고 데이터를 복사한 뒤 그 배열을 가리킵니다.

```go
nums := make([]int, 2, 4)
nums[0], nums[1] = 10, 20

nums = append(nums, 30)
nums = append(nums, 40)
```

위 경우에는 용량이 4이므로 같은 기반 배열 안에서 확장될 가능성이 큽니다.  
하지만 한 번 더 `append()`하면 새 배열이 필요할 수 있습니다.

그래서 슬라이스를 다룰 때는 "이 값이 독립 복사본인지, 같은 배열을 공유하는 뷰인지"를 구분해야 합니다.

### 슬라이싱
---
슬라이싱은 배열이나 슬라이스에서 일부 구간을 잘라 새 슬라이스를 만드는 문법입니다.

```go
slice[시작:끝]
```

여기서 `시작`은 포함되고 `끝`은 포함되지 않습니다.

```go
nums := []int{10, 20, 30, 40, 50}

fmt.Println(nums[1:4]) // [20 30 40]
```

생략도 가능합니다.

```go
fmt.Println(nums[:3]) // [10 20 30]
fmt.Println(nums[2:]) // [30 40 50]
fmt.Println(nums[:])  // [10 20 30 40 50]
```

슬라이싱은 원본 데이터를 복사하지 않고 같은 기반 배열을 공유합니다.

```go
nums := []int{10, 20, 30, 40, 50}
part := nums[1:4]

part[1] = 777

fmt.Println(nums) // [10 20 777 40 50]
```

슬라이스에서 또 슬라이스를 만들 수도 있습니다.

```go
nums := []int{10, 20, 30, 40, 50}
a := nums[1:5]
b := a[1:3]

fmt.Println(a) // [20 30 40 50]
fmt.Println(b) // [30 40]
```

용량은 시작 위치를 기준으로 남은 범위까지 계산됩니다.

```go
nums := []int{10, 20, 30, 40, 50}
part := nums[1:3]

fmt.Println(len(part)) // 2
fmt.Println(cap(part)) // 4
```

`part`는 인덱스 1에서 시작했기 때문에,  
기반 배열 끝까지 총 4칸을 사용할 수 있습니다.

필요하다면 세 번째 인덱스를 사용해 용량까지 제한할 수 있습니다.

```go
nums := []int{10, 20, 30, 40, 50}
part := nums[1:3:3]

fmt.Println(len(part)) // 2
fmt.Println(cap(part)) // 2
```

이 문법은 고급 기능이지만,  
원본과의 의도치 않은 공유 범위를 줄이고 싶을 때 도움이 됩니다.

### 유용한 슬라이싱 기능 활용
---
슬라이스는 단순히 자르는 용도만이 아니라,  
데이터를 다루는 기본 도구처럼 자주 사용됩니다.

가장 많이 쓰는 기능은 `append()`입니다.

```go
nums := []int{1, 2, 3}
nums = append(nums, 4)
nums = append(nums, 5, 6)
```

슬라이스끼리 합칠 때는 `...`를 사용합니다.

```go
a := []int{1, 2}
b := []int{3, 4}

a = append(a, b...)
fmt.Println(a) // [1 2 3 4]
```

복사가 필요할 때는 `copy()`를 사용합니다.

```go
src := []int{10, 20, 30}
dst := make([]int, len(src))

copy(dst, src)
dst[0] = 999

fmt.Println(src) // [10 20 30]
fmt.Println(dst) // [999 20 30]
```

슬라이스끼리 단순 대입만 하면 기반 배열을 공유할 수 있지만,  
`copy()`를 사용하면 독립적인 복사본을 만들 수 있습니다.

앞쪽 요소를 제거하는 패턴도 자주 나옵니다.

```go
queue := []int{10, 20, 30, 40}
queue = queue[1:]

fmt.Println(queue) // [20 30 40]
```

뒤쪽 요소를 제거하는 것도 간단합니다.

```go
stack := []int{10, 20, 30, 40}
stack = stack[:len(stack)-1]

fmt.Println(stack) // [10 20 30]
```

중간 구간만 떼어내서 처리할 수도 있습니다.

```go
nums := []int{10, 20, 30, 40, 50, 60}
middle := nums[2:5]

fmt.Println(middle) // [30 40 50]
```

문자열도 바이트 슬라이스처럼 다룰 수 있지만,  
한글처럼 여러 바이트 문자가 섞이면 `[]byte`와 `[]rune`의 차이를 구분해야 합니다.

```go
text := "Hello"
bytes := []byte(text)
bytes[0] = 'h'

fmt.Println(string(bytes)) // hello
```

처음에는 정수 슬라이스로 감각을 익히고,  
문자열 변환은 바이트와 문자 차이를 이해한 뒤 사용하는 편이 좋습니다.

### 슬라이스 정렬
---
슬라이스는 정렬과 함께 자주 사용됩니다.  
Go에서는 표준 라이브러리로 정렬 기능을 제공합니다.

기본적인 정수 정렬은 `sort.Ints()`를 사용합니다.

```go
nums := []int{30, 10, 50, 20, 40}
sort.Ints(nums)

fmt.Println(nums) // [10 20 30 40 50]
```

문자열 슬라이스는 `sort.Strings()`로 정렬합니다.

```go
names := []string{"go", "java", "c"}
sort.Strings(names)

fmt.Println(names) // [c go java]
```

내림차순 정렬도 가능합니다.

```go
nums := []int{30, 10, 50, 20, 40}
sort.Sort(sort.Reverse(sort.IntSlice(nums)))

fmt.Println(nums) // [50 40 30 20 10]
```

구조체처럼 기준이 복잡한 경우에는 `sort.Slice()`를 사용할 수 있습니다.

```go
people := []struct {
	Name string
	Age  int
}{
	{"Kim", 30},
	{"Lee", 25},
	{"Park", 28},
}

sort.Slice(people, func(i, j int) bool {
	return people[i].Age < people[j].Age
})
```

정렬 함수 대부분은 원본 슬라이스를 직접 바꿉니다.  
즉 정렬 결과를 새 값으로 반환하는 것이 아니라, 기존 슬라이스 순서를 재배치합니다.

그래서 원본 순서를 보존해야 한다면 먼저 복사본을 만들어야 합니다.

### 핵심 요약
---
- 슬라이스는 배열의 일부를 가리키는 가변 길이 뷰입니다.
- 슬라이스는 길이와 용량을 가지며, `len()`과 `cap()`으로 확인할 수 있습니다.
- 슬라이스는 기반 배열을 공유할 수 있으므로 요소 변경이 서로 영향을 줄 수 있습니다.
- `append()`는 남은 용량이 있으면 같은 배열을 재사용하고, 부족하면 새 배열을 만들 수 있습니다.
- 슬라이싱은 `시작:끝` 형태이며 시작은 포함, 끝은 제외합니다.
- 독립 복사본이 필요하면 `copy()`를 사용해야 합니다.
- 정수, 문자열, 사용자 정의 기준 정렬까지 표준 라이브러리로 처리할 수 있습니다.
