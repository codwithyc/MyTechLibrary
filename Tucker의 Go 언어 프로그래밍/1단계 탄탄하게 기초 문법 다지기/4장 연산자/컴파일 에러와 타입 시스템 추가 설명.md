## 컴파일 에러와 타입 시스템 추가 설명

연산자를 공부하다 보면 `int`와 `int64`를 더하려고 했는데 에러가 나거나,  
정수끼리 나눴는데 소수점이 사라지는 상황을 자주 만납니다.

이때 중요한 것은 연산자 자체보다 Go가 코드를 실행하기 전에 타입을 얼마나 엄격하게 검사하는지 이해하는 것입니다.

### Go는 실행 전에 많은 실수를 막는다
---

Go는 컴파일 언어입니다.  
코드를 실행하기 전에 컴파일러가 문법, 타입, 사용하지 않는 변수 등을 먼저 확인합니다.

```go
var a int = 10
var b int64 = 20

// fmt.Println(a + b)
```

위 코드는 실행되기 전에 컴파일 단계에서 막힙니다.  
Go는 `int`와 `int64`를 자동으로 섞어 계산하지 않기 때문입니다.

처음에는 불편하지만, 이 엄격함 덕분에 값의 의미와 범위를 더 분명히 생각하게 됩니다.

### 컴파일 에러는 공부 자료다
---

Go를 공부할 때 컴파일 에러를 단순히 실패 메시지로 보지 않는 것이 좋습니다.  
컴파일 에러는 Go가 어떤 규칙을 요구하는지 알려주는 가장 직접적인 피드백입니다.

예를 들어 다음과 같은 메시지를 만날 수 있습니다.

```text
invalid operation: a + b (mismatched types int and int64)
```

이 메시지는 두 가지를 알려줍니다.

- `a + b` 연산이 문제입니다.
- 두 값의 타입이 `int`와 `int64`로 서로 다릅니다.

에러를 읽을 때는 "어디서", "어떤 규칙 때문에", "어떤 타입끼리" 문제가 생겼는지 나누어 보면 좋습니다.

### Go는 암시적 변환을 거의 하지 않는다
---

일부 언어는 정수와 실수를 섞어 계산할 때 자동으로 타입을 바꿔줍니다.  
Go는 이런 자동 변환을 거의 하지 않습니다.

```go
var count int = 3
var ratio float64 = 1.5

// fmt.Println(count * ratio)
fmt.Println(float64(count) * ratio)
```

Go에서는 변환 의도를 코드에 직접 적어야 합니다.  
이 방식은 조금 길어 보이지만, 어떤 값의 기준으로 계산하는지 명확하게 드러냅니다.

### 사용하지 않는 변수도 컴파일 에러다
---

Go를 처음 배울 때 자주 만나는 에러 중 하나가 사용하지 않는 변수입니다.

```go
func main() {
    result := 10 + 20
}
```

`result`를 선언했지만 사용하지 않았기 때문에 컴파일 에러가 납니다.  
Go는 불필요한 코드가 쌓이는 것을 초반부터 막습니다.

잠깐 값을 버려야 한다면 blank identifier인 `_`를 사용할 수 있습니다.

```go
_ = result
```

다만 `_`는 임시로 쓰는 도구이지, 의미 없는 코드를 숨기기 위한 습관이 되어서는 안 됩니다.

### go build와 go run의 피드백 차이
---

학습 중에는 `go run`을 자주 사용합니다.

```bash
go run main.go
```

`go run`은 컴파일한 뒤 바로 실행합니다.  
반면 `go build`는 실행 파일을 만들면서 컴파일 가능 여부를 확인합니다.

```bash
go build
```

프로그램을 실행하지 않고도 컴파일 에러만 확인하고 싶다면 `go build`가 유용합니다.  
연산자나 타입 실험을 할 때도 실행 전에 코드가 유효한지 확인하는 좋은 습관이 됩니다.

### 정리
---

연산자를 배우는 과정은 사실 Go의 타입 시스템을 처음으로 강하게 체감하는 과정입니다.

정리하면 다음과 같습니다.

- Go는 실행 전에 타입이 맞는지 엄격하게 검사합니다.
- 컴파일 에러는 Go의 규칙을 알려주는 학습 자료입니다.
- Go는 암시적 타입 변환을 거의 하지 않습니다.
- 사용하지 않는 변수도 컴파일 에러로 막습니다.
- `go build`는 실행하지 않고 컴파일 가능 여부를 확인할 때 유용합니다.

Go를 잘 익히려면 에러를 빨리 없애는 것보다, 에러 메시지가 말하는 타입 규칙을 읽는 습관이 더 중요합니다.
