## 값 복사와 참조처럼 보이는 값 추가 설명

배열을 배우면 Go에서 값이 복사된다는 사실을 처음 강하게 체감하게 됩니다.  
하지만 이후 슬라이스, 맵, 포인터를 배우면 어떤 값은 복사되었는데도 원본이 바뀌는 것처럼 보입니다.

이번 추가 설명은 배열 문법 자체가 아니라, Go를 공부할 때 계속 따라오는 값 복사 감각을 정리합니다.

### Go는 기본적으로 값을 복사한다
---

Go에서 대입은 기본적으로 값을 복사합니다.

```go
a := 10
b := a
b = 20

fmt.Println(a) // 10
fmt.Println(b) // 20
```

배열도 값입니다.

```go
a := [3]int{1, 2, 3}
b := a
b[0] = 100

fmt.Println(a) // [1 2 3]
fmt.Println(b) // [100 2 3]
```

이 규칙을 먼저 이해해야 이후 자료구조가 덜 헷갈립니다.

### 슬라이스는 배열 자체가 아니라 배열을 보는 창이다
---

슬라이스는 배열과 다르게 동작하는 것처럼 보입니다.

```go
a := []int{1, 2, 3}
b := a
b[0] = 100

fmt.Println(a) // [100 2 3]
```

이 코드만 보면 슬라이스는 복사되지 않는 것처럼 느껴질 수 있습니다.  
하지만 실제로는 슬라이스 값이 복사됩니다.

슬라이스 값 안에는 내부 배열을 가리키는 정보가 들어 있습니다.  
그래서 슬라이스 값은 복사되었지만, 두 슬라이스가 같은 내부 배열을 바라볼 수 있습니다.

### 맵도 값은 복사되지만 내부 데이터는 공유된다
---

맵도 비슷합니다.

```go
a := map[string]int{"go": 1}
b := a
b["go"] = 100

fmt.Println(a["go"]) // 100
```

맵 변수 자체는 값이지만, 내부 데이터 구조를 가리키는 핸들처럼 동작합니다.  
그래서 대입 후 한쪽에서 값을 바꾸면 다른 쪽에서도 바뀐 것처럼 보입니다.

### 값이 복사되는지, 내부 데이터를 공유하는지 구분하자
---

Go를 공부할 때는 "복사냐 참조냐"를 너무 단순하게 나누기보다 다음 질문을 해보는 것이 좋습니다.

- 이 값 자체가 모든 데이터를 직접 담고 있는가?
- 아니면 내부 데이터를 가리키는 정보를 담고 있는가?
- 대입했을 때 원본과 새 변수가 같은 내부 데이터를 바라보는가?

배열과 구조체는 값 전체가 복사되는 성격이 강합니다.  
슬라이스와 맵은 값은 복사되지만 내부 데이터를 공유할 수 있습니다.

### 함수에 넘길 때도 같은 질문을 한다
---

함수 호출도 값 전달입니다.

```go
func ChangeArray(arr [3]int) {
    arr[0] = 100
}
```

배열은 전체가 복사되므로 원본이 바뀌지 않습니다.

```go
func ChangeSlice(nums []int) {
    nums[0] = 100
}
```

슬라이스는 값이 복사되지만 내부 배열을 공유하므로 요소 변경이 원본에 보일 수 있습니다.

이 차이는 Go 자료구조를 이해하는 핵심입니다.

### 정리
---

배열 장에서 가장 중요한 추가 감각은 값 복사입니다.

정리하면 다음과 같습니다.

- Go의 대입과 함수 호출은 기본적으로 값을 복사합니다.
- 배열은 값 전체가 복사됩니다.
- 슬라이스는 값은 복사되지만 내부 배열을 공유할 수 있습니다.
- 맵도 값은 복사되지만 내부 데이터가 공유됩니다.
- "복사되었는가"와 "내부 데이터를 공유하는가"를 나누어 생각해야 합니다.

이 감각이 잡히면 이후 슬라이스, 맵, 포인터를 배울 때 Go의 동작이 훨씬 자연스럽게 이해됩니다.
