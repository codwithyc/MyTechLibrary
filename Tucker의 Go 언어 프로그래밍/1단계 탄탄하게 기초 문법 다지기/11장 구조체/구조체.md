## 구조체

### 구조체란?
---
구조체는 서로 관련 있는 여러 값을 하나로 묶어 새로운 타입을 만드는 문법입니다.  
배열이 같은 타입의 값을 여러 개 묶는 도구라면, 구조체는 서로 다른 타입의 값도 함께 묶을 수 있습니다.

예를 들어 사람 한 명을 표현한다고 해보겠습니다.

```go
name := "gopher"
age := 10
height := 140.5
```

이 값들은 모두 한 사람에 대한 정보입니다.  
하지만 변수만 따로 있으면 이 값들이 서로 관련 있다는 사실이 코드에 잘 드러나지 않습니다.

구조체를 사용하면 하나의 타입으로 묶을 수 있습니다.

```go
type Person struct {
	Name   string
	Age    int
	Height float64
}
```

이제 `Person`은 이름, 나이, 키를 가진 하나의 타입입니다.

### 선언 및 기본 사용
---
구조체는 `type`과 `struct` 키워드로 선언합니다.

```go
type 타입이름 struct {
	필드이름 타입
	필드이름 타입
}
```

예시를 보겠습니다.

```go
type Student struct {
	Name  string
	Score int
}
```

구조체 안에 들어 있는 각각의 값을 필드라고 부릅니다.  
`Student` 구조체에는 `Name` 필드와 `Score` 필드가 있습니다.

구조체 변수를 만들고 필드에 접근할 때는 점 `.`을 사용합니다.

```go
var student Student

student.Name = "gopher"
student.Score = 95

fmt.Println(student.Name)
fmt.Println(student.Score)
```

구조체는 사용자가 직접 만드는 타입입니다.  
즉 Go가 기본으로 제공하는 `int`, `string`, `bool`처럼,  
프로그램에 필요한 개념을 타입으로 표현할 수 있게 해줍니다.

### 구조체 변수 초기화
---
구조체는 여러 방식으로 초기화할 수 있습니다.

필드 이름을 지정하는 방식입니다.

```go
student := Student{
	Name:  "gopher",
	Score: 95,
}
```

이 방식은 어떤 값이 어떤 필드에 들어가는지 명확해서 가장 많이 사용됩니다.

필드 순서대로 값을 넣을 수도 있습니다.

```go
student := Student{"gopher", 95}
```

하지만 이 방식은 필드 순서가 바뀌면 실수하기 쉽습니다.  
처음에는 필드 이름을 적는 방식을 사용하는 것이 좋습니다.

값을 지정하지 않은 필드는 각 타입의 기본값으로 초기화됩니다.

```go
student := Student{Name: "gopher"}

fmt.Println(student.Score) // 0
```

`int`의 기본값은 `0`, `string`의 기본값은 빈 문자열 `""`입니다.

### 구조체를 포함하는 구조체
---
구조체 안에 다른 구조체를 필드로 넣을 수 있습니다.  
이렇게 하면 더 큰 개념을 작은 개념의 조합으로 표현할 수 있습니다.

```go
type Address struct {
	City  string
	Street string
}

type User struct {
	Name    string
	Age     int
	Address Address
}
```

사용할 때는 필드를 이어서 접근합니다.

```go
user := User{
	Name: "gopher",
	Age:  10,
	Address: Address{
		City:   "Seoul",
		Street: "Go-ro",
	},
}

fmt.Println(user.Address.City)
```

구조체를 구조체 안에 넣으면 데이터의 관계를 더 자연스럽게 표현할 수 있습니다.  
예를 들어 사용자는 주소를 가지고 있고, 주문은 상품 목록과 배송지를 가질 수 있습니다.

### 구조체 크기
---
구조체는 여러 필드를 묶은 타입이므로, 구조체 변수는 각 필드의 값을 저장할 공간을 가집니다.  
구조체의 크기는 필드 타입과 배치 방식에 따라 결정됩니다.

```go
type Sample struct {
	A int8
	B int64
	C int8
}
```

단순히 `int8 + int64 + int8` 크기로만 생각하면 안 됩니다.  
컴퓨터는 데이터를 더 효율적으로 읽기 위해 메모리 정렬을 사용합니다.  
그래서 구조체에는 필드 사이에 사용하지 않는 빈 공간이 생길 수 있습니다.

처음부터 구조체 크기를 깊게 최적화할 필요는 없습니다.  
다만 구조체는 실제 메모리에 값들이 배치되는 타입이라는 점을 기억하면 됩니다.

나중에 성능이나 메모리 사용량을 다룰 때 구조체 필드 순서가 중요해질 수 있습니다.

### 프로그래밍에서 구조체의 역할
---
구조체는 Go에서 매우 중요한 문법입니다.  
Go에는 클래스가 없지만, 구조체를 사용해 프로그램의 핵심 데이터를 표현합니다.

예를 들어 게임 캐릭터를 구조체로 표현할 수 있습니다.

```go
type Character struct {
	Name  string
	Level int
	HP    int
}
```

웹 서비스라면 사용자를 구조체로 표현할 수 있습니다.

```go
type User struct {
	ID    int
	Email string
	Name  string
}
```

구조체를 잘 설계하면 코드가 훨씬 읽기 쉬워집니다.  
왜냐하면 프로그램에서 다루는 개념이 타입으로 드러나기 때문입니다.

좋은 구조체를 만들 때는 다음 질문을 해보면 좋습니다.

- 이 값들은 하나의 개념으로 묶일 수 있는가?
- 필드 이름만 보고 의미를 알 수 있는가?
- 불필요하게 많은 책임을 가진 구조체는 아닌가?

### 핵심 요약
---
- 구조체는 서로 관련 있는 여러 값을 하나의 타입으로 묶는 문법입니다.
- `type 이름 struct { ... }` 형태로 선언합니다.
- 구조체 안의 값을 필드라고 부릅니다.
- 필드에는 점 `.`으로 접근합니다.
- 구조체 안에 다른 구조체를 포함할 수 있습니다.
- Go에서는 구조체가 프로그램의 주요 데이터를 표현하는 핵심 도구입니다.
