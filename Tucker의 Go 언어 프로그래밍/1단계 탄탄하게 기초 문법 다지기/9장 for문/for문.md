## for문

### for문이란?
---
반복문은 같은 코드를 여러 번 실행할 때 사용하는 문법입니다.  
Go에는 반복문이 `for` 하나뿐입니다.

다른 언어에는 `while`, `do while` 같은 반복문이 따로 있지만,  
Go는 `for`문 하나로 여러 형태의 반복을 표현합니다.

```go
for 초기문; 조건문; 후처리 {
	반복할 코드
}
```

예를 들어 1부터 5까지 출력하는 코드는 다음과 같습니다.

```go
for i := 1; i <= 5; i++ {
	fmt.Println(i)
}
```

이 코드는 다음 순서로 실행됩니다.

1. `i := 1`을 한 번 실행합니다.
2. `i <= 5` 조건을 검사합니다.
3. 조건이 `true`이면 본문을 실행합니다.
4. `i++`를 실행합니다.
5. 다시 조건을 검사합니다.
6. 조건이 `false`가 되면 반복을 끝냅니다.

### for문 동작 원리
---
`for`문의 세 부분을 나누어 보면 이해하기 쉽습니다.

```go
for i := 0; i < 3; i++ {
	fmt.Println(i)
}
```

- 초기문: `i := 0`
    반복을 시작하기 전에 한 번 실행됩니다.
- 조건문: `i < 3`
    반복을 계속할지 결정합니다.
- 후처리: `i++`
    반복 본문이 한 번 끝날 때마다 실행됩니다.

실행 결과는 다음과 같습니다.

```text
0
1
2
```

조건문이 처음부터 `false`이면 본문은 한 번도 실행되지 않습니다.

```go
for i := 0; i < 0; i++ {
	fmt.Println(i)
}
```

반복문은 조건을 잘못 작성하면 끝나지 않을 수 있습니다.  
이런 반복을 무한 반복이라고 합니다.

```go
for {
	fmt.Println("계속 반복합니다.")
}
```

무한 반복 자체가 나쁜 것은 아닙니다.  
서버처럼 계속 실행되어야 하는 프로그램에서는 의도적으로 사용하기도 합니다.  
다만 종료 조건이 필요한 반복에서 실수로 무한 반복이 되지 않도록 주의해야 합니다.

### continue와 break
---
반복문 안에서는 `continue`와 `break`로 흐름을 제어할 수 있습니다.

`continue`는 현재 반복을 건너뛰고 다음 반복으로 넘어갑니다.

```go
for i := 1; i <= 5; i++ {
	if i%2 == 0 {
		continue
	}

	fmt.Println(i)
}
```

위 코드는 짝수일 때 출력을 건너뛰므로 홀수만 출력합니다.

`break`는 반복문을 즉시 끝냅니다.

```go
for i := 1; i <= 10; i++ {
	if i == 5 {
		break
	}

	fmt.Println(i)
}
```

위 코드는 `i`가 5가 되는 순간 반복을 끝냅니다.

정리하면 다음과 같습니다.

- `continue`
    이번 반복만 건너뛰고 다음 반복으로 이동합니다.
- `break`
    반복문 자체를 끝냅니다.

### 조건만 있는 for문
---
Go의 `for`문은 초기문과 후처리를 생략하고 조건만 사용할 수 있습니다.  
다른 언어의 `while`문처럼 사용할 수 있습니다.

```go
count := 3

for count > 0 {
	fmt.Println(count)
	count--
}
```

Go에는 `while` 키워드가 없지만, 이 형태가 사실상 `while` 역할을 합니다.

### 중첩 for문
---
반복문 안에 반복문을 넣을 수 있습니다.  
이것을 중첩 for문이라고 합니다.

구구단을 출력하는 예시를 보겠습니다.

```go
for dan := 2; dan <= 9; dan++ {
	for num := 1; num <= 9; num++ {
		fmt.Printf("%d x %d = %d\n", dan, num, dan*num)
	}
}
```

바깥 반복문은 단을 나타내고, 안쪽 반복문은 곱하는 수를 나타냅니다.  
바깥 반복이 한 번 실행될 때마다 안쪽 반복은 처음부터 끝까지 실행됩니다.

중첩 반복문은 표, 좌표, 이중 배열처럼 두 방향으로 반복해야 할 때 자주 사용됩니다.  
하지만 깊이가 너무 깊어지면 코드가 복잡해지므로 함수로 나누는 것도 고려해야 합니다.

### 중첩 for문과 break, 레이블
---
중첩 반복문에서 `break`를 사용하면 가장 가까운 반복문만 종료됩니다.

```go
for i := 0; i < 3; i++ {
	for j := 0; j < 3; j++ {
		if j == 1 {
			break
		}
		fmt.Println(i, j)
	}
}
```

위 코드에서 `break`는 안쪽 `for`문만 끝냅니다.  
바깥 반복문은 계속 실행됩니다.

바깥 반복문까지 한 번에 끝내고 싶다면 레이블을 사용할 수 있습니다.

```go
Outer:
for i := 0; i < 3; i++ {
	for j := 0; j < 3; j++ {
		if i == 1 && j == 1 {
			break Outer
		}
		fmt.Println(i, j)
	}
}
```

레이블은 강력하지만 남용하면 흐름을 따라가기 어려워집니다.  
복잡한 중첩 반복이 생기면 레이블을 쓰기 전에 함수로 분리할 수 있는지 먼저 생각해보는 것이 좋습니다.

### 핵심 요약
---
- Go의 반복문은 `for` 하나입니다.
- `for 초기문; 조건문; 후처리` 형태로 반복을 제어합니다.
- 조건만 사용하면 다른 언어의 `while`문처럼 사용할 수 있습니다.
- `continue`는 현재 반복을 건너뛰고, `break`는 반복문을 끝냅니다.
- 중첩 반복문에서는 `break`가 가장 가까운 반복문만 종료합니다.
- 레이블을 사용하면 바깥 반복문까지 제어할 수 있지만, 필요한 경우에만 사용하는 것이 좋습니다.
